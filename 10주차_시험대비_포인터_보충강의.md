## 0. 들어가기 — 포인터는 도대체 왜 어려운가?

포인터가 어려운 이유는 단 하나, **"값"과 "주소"를 동시에 머릿속에 그려야 하기 때문**이다.

> 🍳 **주방 비유 — 반찬통과 라벨**
> - 변수 = **반찬통** (안에 값이 들어 있음)
> - 변수의 주소 = **반찬통이 놓인 자리 번호** (예: "냉장고 3번 칸")
> - 포인터 = **다른 반찬통의 자리 번호가 적힌 라벨**
>
> 라벨 자체도 작은 통에 담겨 있다(포인터도 변수다). 라벨에 적힌 번호로 찾아가면 진짜 반찬통이 나온다.

이 그림을 머릿속에 박아 두고 시작하자.

## 1. 변수와 메모리 — 모든 것의 출발점

### 1.1 메모리는 바이트 단위로 번호가 매겨진 거대한 선반

```
주소:    0     1     2     3     4     5     ...   (바이트 단위)
       ┌─────┬─────┬─────┬─────┬─────┬─────┐
       │     │     │     │     │     │     │  ← 메모리 (RAM)
       └─────┴─────┴─────┴─────┴─────┴─────┘
```

### 1.2 자료형에 따라 변수가 차지하는 칸 수가 다르다

| 자료형 | 차지하는 바이트 |
|--------|-----------------|
| `char`   | 1 |
| `short`  | 2 |
| `int`    | 4 |
| `long`   | 4 또는 8 (시스템 따라) |
| `float`  | 4 |
| `double` | 8 |

> 🍳 **반찬통의 크기 차이**
> `char`는 작은 종지, `int`는 보통 그릇, `double`은 큰 대접.

### 1.3 변수의 주소 — `&` 연산자

```c
#include <stdio.h>

int main(void) {
    int   i = 10;
    char  c = 69;
    float f = 12.3;

    printf("i의 주소: %p\n", &i);   // 변수 i가 놓인 자리 번호
    printf("c의 주소: %p\n", &c);
    printf("f의 주소: %p\n", &f);
    return 0;
}
```

> ⚠️ 주소는 **실행할 때마다 달라진다.** (OS가 매번 다른 자리를 할당하기 때문)
> 형식 지정자는 `%p` (pointer).

> **문제:** 다음 변수들의 **값**과 **주소**를 모두 출력하시오.
> `int a = 100;`, `double b = 3.14;`, `char c = 'K';`

<details>
<summary>👉 정답 코드</summary>

```c
}
```

**관찰 포인트:** 자료형 크기가 다르기 때문에 인접한 변수 주소들 간격이 다르게 보일 수 있다.

</details>

## 2. 포인터의 선언과 초기화

### 2.1 기본 문법

```c
int* p;          // int를 가리키는 포인터
int *p;          // 같은 의미 (* 위치는 자유)
int * p;         // 같은 의미

// 변수의 주소를 대입
int i = 10;
int *p = &i;     // p는 i의 주소를 담는다 → p는 i를 가리킴
```

### 2.2 ★ 시험 단골 함정 — 한 줄에 여러 포인터 선언

```c
int *p1, p2, p3;     // ❌ p1만 포인터, p2와 p3는 그냥 int!
int *p1, *p2, *p3;   // ✅ 셋 다 포인터
```

> 🍳 **암기 팁:** `*`는 변수 이름에 붙는다고 생각하자.
> "별표 없는 놈은 그냥 일반 변수다."

### 2.3 다양한 자료형의 포인터

```c
char   c = 'A';
float  f = 36.5;
double d = 3.141592;

char   *pc = &c;       // 문자형 포인터
float  *pf = &f;       // 실수형 포인터
double *pd = &d;       // 더블형 포인터
```

**시험 포인트:** **포인터 타입과 가리키는 변수의 타입은 일치해야 한다.**

```c
int i;
double *pd = &i;       // ❌ 타입 불일치 (경고/오류)
```

### 2.4 NULL 포인터

```c
int *p = NULL;          // "아직 아무것도 안 가리킴"의 표준 표현
```

> ⚠️ **꼭 NULL로 초기화하는 이유**
> 초기화 안 된 포인터(`int *p;`)는 **쓰레기 주소**를 가리킨다.
> 그 상태에서 `*p = 100;`을 하면 **OS가 보호하는 영역을 침범** → 프로그램이 크래시.

## 3. 두 핵심 연산자 — `&`와 `*`

### 3.1 정리 표 (반드시 외우자)

| 연산자 | 이름 | 의미 | 예시 |
|--------|------|------|------|
| `&` | 주소 연산자 (address-of) | 변수의 **주소**를 얻는다 | `&i` → i의 주소 |
| `*` | 간접 참조 연산자 (dereference) | 포인터가 가리키는 **값**을 얻는다 | `*p` → p가 가리키는 값 |

### 3.2 핵심 예제

```c
#include <stdio.h>

int main(void) {
    int i = 3000;
    int *p = NULL;

    p = &i;                           // p가 i를 가리킴

    printf("p   = %p\n", p);          // i의 주소
    printf("&i  = %p\n", &i);         // 같은 값
    printf("i   = %d\n", i);          // 3000
    printf("*p  = %d\n", *p);         // 3000 (p가 가리키는 값)

    *p = 20;                          // ★ p를 통해 i 값을 변경
    printf("i   = %d\n", i);          // 20

    return 0;
}
```

> 🍳 **`&`와 `*`는 반대 연산**
> `&i`는 "값 → 주소", `*p`는 "주소 → 값".
> 그래서 `*(&i)`는 다시 `i` 자신과 같다.

### 3.3 그림으로 외우기

```
   변수 i                포인터 p
┌─────────────┐      ┌──────────────────┐
│  3000       │ ←──  │  (i의 주소)       │
│ 주소: 0x100  │      │  주소: 0x200      │
└─────────────┘      └──────────────────┘

&i  → 0x100  (i의 주소)
*p  → 3000   (p가 가리키는 곳의 값)
*p = 20;  →  i가 20으로 바뀜
```

---

### 🛠️ 학생 실습 #2 — `&`, `*` 손으로 추적

다음 코드의 출력을 **종이에 먼저** 예측해 본 뒤 실행해 보시오.

```c
#include <stdio.h>
int main(void) {
    int x = 10, y = 20;
    int *p;

    p = &x;
    *p = 100;       // (1) 무엇이 바뀌는가?

    p = &y;
    *p = 200;       // (2) 무엇이 바뀌는가?

    printf("x = %d, y = %d\n", x, y);
    return 0;
}
```

<details>
<summary>👉 예상 결과</summary>

```
x = 100, y = 200
```

**해설:** p가 처음엔 x를 가리키므로 `*p = 100`은 x를 변경. 이후 p가 y로 옮겨갔으므로 `*p = 200`은 y를 변경.

</details>

## 4. 포인터 연산 (★ 시험 핵심)

### 4.1 가능한 연산

- `p + n`, `p - n` (정수 덧셈/뺄셈)
- `p++`, `p--`
- 포인터끼리 뺄셈 (같은 배열 안에서)
- 포인터끼리 비교 (`==`, `!=`, `<`, `>`)

### 4.2 ★ 가장 중요한 한 문장

> **`p + 1`은 1바이트 더해지는 게 아니라, `sizeof(*p)`만큼 더해진다.**

```c
char   *pc = (char  *)10000;
int    *pi = (int   *)10000;
double *pd = (double *)10000;

pc + 1  →  10001  (1바이트 이동)
pi + 1  →  10004  (4바이트 이동)
pd + 1  →  10008  (8바이트 이동)
```

> 🍳 **왜 그럴까?**
> 포인터는 "다음 원소"를 가리키는 것이 목적이다.
> `int` 배열에서 다음 원소는 4바이트 떨어져 있고, `double` 배열은 8바이트 떨어져 있다.
> → 컴파일러가 알아서 자료형 크기만큼 곱해서 더한다.

### 4.3 ★★ 시험 빈출 — `*p++` vs `(*p)++`

| 수식 | 의미 |
|------|------|
| `*p++` | p가 가리키는 값을 가져온 후 **p를 다음 칸으로 이동** |
| `(*p)++` | p가 가리키는 **값**을 1 증가 |
| `*++p` | p를 먼저 이동시킨 뒤 그 위치의 값을 가져옴 |
| `++*p` | p가 가리키는 값을 1 증가시킨 뒤 그 값을 사용 |

**실습 코드로 확인:**
```c
#include <stdio.h>
int main(void) {
    int i = 10;
    int *pi = &i;

    printf("i = %d, pi = %p\n", i, pi);   // i = 10

    (*pi)++;                              // pi가 가리키는 값(i)을 +1
    printf("i = %d, pi = %p\n", i, pi);   // i = 11, pi는 그대로

    *pi++;                                // pi가 가리키는 값을 가져온 뒤 pi를 +1
    printf("i = %d, pi = %p\n", i, pi);   // i = 11, pi는 다음 칸으로 이동

    return 0;
}
```

> ⚠️ **시험 출제 패턴:** "`*p++`와 `(*p)++`의 차이를 설명하시오." → **외워두자.**

### 4.4 위험성 경고

> 포인터는 우리 마음대로 증감 가능하다. 하지만 **잘못된 위치를 가리키면 OS 영역을 침범**한다.
> → 포인터 연산은 **반드시 자기 배열 안에서만** 한다.

---

### 🛠️ 학생 실습 #3 — `*p++` 결과 예측

다음 코드의 출력을 예측한 뒤 실행해 보시오.

```c
#include <stdio.h>
int main(void) {
    int arr[5] = {10, 20, 30, 40, 50};
    int *p = arr;

    printf("%d ", *p++);    // (1)
    printf("%d ", *p);      // (2)
    printf("%d ", *p);      // (3)
    printf("%d ", *++p);    // (4)
    printf("%d\n", *p);     // (5)
    return 0;
}
```

<details>
<summary>👉 예상 결과</summary>

```
10 20 20 30 30
```

**해설:**
- (1) `*p++`: p가 가리키는 값 10을 출력하고 p는 arr[1]로 이동
- (2),(3) p는 arr[1]을 가리키므로 20
- (4) `*++p`: p를 먼저 arr[2]로 옮기고 그 값 30 출력
- (5) p는 arr[2] 위치에 있으므로 30

</details>

## 5. 포인터와 함수 (★★★ 시험 최빈출)

### 5.1 두 가지 인수 전달 방식

| 방식 | 무엇이 전달되나 | 원본 변경? |
|------|------------------|-----------|
| **값에 의한 호출 (call by value)** | 변수의 복사본 | 불가능 |
| **참조에 의한 호출 (call by reference)** | 변수의 **주소** | 가능 |

> 🍳 **주방 비유**
> - 값에 의한 호출: 반찬통 내용을 **복사한 새 통**을 보낸다. 받은 사람이 어떻게 바꾸든 원본은 안 바뀜.
> - 참조에 의한 호출: 원본 반찬통이 놓인 **자리 번호(주소)**를 알려준다. 받은 사람이 직접 그 자리로 찾아가 내용을 바꿈.

### 5.2 ★ 단골 예제 — `swap` 함수

**❌ 잘못된 버전 (값에 의한 호출 → 안 바뀜)**
```c
void swap(int x, int y) {        // 복사본을 받음
    int tmp = x;
    x = y;
    y = tmp;
    // 여기 안에서만 바뀜, 함수 끝나면 사라짐
}

int main(void) {
    int a = 100, b = 200;
    swap(a, b);
    printf("a=%d b=%d\n", a, b);  // a=100 b=200 (안 바뀜!)
}
```

**✅ 올바른 버전 (참조에 의한 호출)**
```c
void swap(int *px, int *py) {    // 주소를 받음
    int tmp = *px;
    *px = *py;
    *py = tmp;
}

int main(void) {
    int a = 100, b = 200;
    swap(&a, &b);                 // 주소를 넘김
    printf("a=%d b=%d\n", a, b);  // a=200 b=100 ✓
}
```

### 5.3 `scanf`도 같은 원리

```c
int n;
scanf("%d", &n);    // ★ n의 주소를 넘긴다 — 그래야 scanf가 n에 값을 채울 수 있음
```

> 시험 단골: "왜 `scanf`에서는 변수 앞에 `&`를 붙이는가?" → **scanf가 변수에 값을 저장하려면 변수의 주소를 알아야 하기 때문**.

### 5.4 함수가 여러 값을 반환해야 할 때

C 함수는 `return`으로 값을 **하나만** 돌려준다. 두 개 이상을 돌려주려면 **포인터를 통해** 한다.

```c
// 직선의 기울기와 y절편을 동시에 계산
int get_line_parameter(int x1, int y1, int x2, int y2,
                       float *slope, float *yintercept) {
    if (x1 == x2) return -1;       // 수직선 → 기울기 정의 불가
    *slope      = (float)(y2 - y1) / (x2 - x1);
    *yintercept = y1 - (*slope) * x1;
    return 0;
}

int main(void) {
    float s, y;
    if (get_line_parameter(3, 3, 6, 6, &s, &y) == -1)
        printf("에러\n");
    else
        printf("기울기=%.2f, y절편=%.2f\n", s, y);
}
```

### 5.5 ★ `const`로 원본 보호

함수가 **포인터를 받지만 값을 바꾸지 않을** 것임을 약속하려면 `const`를 붙인다.

```c
void print_value(const int *p) {
    printf("%d\n", *p);
    // *p = 0;       // ❌ 컴파일 에러 — const가 변경을 막음
}
```

### 5.6 ⚠️ **절대 하지 말 것 — 지역변수 주소 반환**

```c
int* bad_function(int x, int y) {
    int result = x + y;
    return &result;       // ❌ 위험! result는 함수가 끝나면 사라짐
}
```

> 함수가 끝나면 지역변수는 사라진다(스택에서 회수).
> 그 주소를 반환하면, 호출자는 **이미 없는 변수**를 가리키는 포인터를 받게 된다.

---

### 🛠️ 학생 실습 #4 — 두 값을 동시에 반환하는 함수

> **문제:** 정수 배열을 받아 **최소값과 최대값을 동시에** 포인터로 반환하는 함수를 작성하시오.
>
> ```c
> void find_min_max(const int a[], int n, int *min_out, int *max_out);
> ```

<details>
<summary>👉 정답 코드</summary>

```c
#include <stdio.h>
#define SIZE 8

void find_min_max(const int a[], int n, int *min_out, int *max_out) {
    *min_out = *max_out = a[0];
    for (int i = 1; i < n; i++) {
        if (a[i] < *min_out) *min_out = a[i];
        if (a[i] > *max_out) *max_out = a[i];
    }
}

int main(void) {
    int arr[SIZE] = {12, 45, 7, 89, 23, 56, 3, 67};
    int min_v, max_v;

    find_min_max(arr, SIZE, &min_v, &max_v);
    printf("최소 = %d, 최대 = %d\n", min_v, max_v);
    return 0;
}
```

**예상 출력:** `최소 = 3, 최대 = 89`

**시험 포인트:** 호출 시 `&min_v`, `&max_v`로 **주소**를 넘긴다. 함수 안에서는 `*min_out`, `*max_out`로 **값에 접근**한다.

</details>

## 6. 포인터와 배열 (★★★ 시험 최빈출)

### 6.1 핵심 한 줄

> **배열 이름은 그 자체가 첫 원소의 주소다.** (즉, 포인터처럼 동작한다)

```c
int a[] = {10, 20, 30, 40, 50};

a       // == &a[0]  (첫 원소의 주소)
a + 1   // == &a[1]
a + 2   // == &a[2]
*a      // == a[0]   == 10
*(a+1)  // == a[1]   == 20
```

### 6.2 4가지 같은 표기법

```c
int a[5] = {10, 20, 30, 40, 50};
int *p = a;            // 배열 이름을 그대로 대입 가능

// 다음 네 가지는 모두 동일!
a[i]       // 표준 배열 표기
*(a + i)   // 배열을 포인터로
p[i]       // 포인터를 배열처럼
*(p + i)   // 포인터 + offset
```

> 🍳 **이걸 처음 보면 신기하지만 익숙해져야 한다.**
> 시험에서는 "다음 중 `a[2]`와 같은 것을 모두 고르시오" 같은 문제가 자주 나온다.
> → `*(a+2)`, `p[2]`, `*(p+2)` 모두 같다.

### 6.3 배열 vs 포인터의 차이점 (★ 시험 함정)

| 항목 | 배열 (`int a[5]`) | 포인터 (`int *p`) |
|------|--------------------|--------------------|
| 메모리 할당 | 자동 (5×4=20바이트) | 포인터 자체만 (8바이트) |
| `sizeof` | 20 (전체 크기) | 8 (포인터 크기) |
| 주소 변경 | 불가 (`a = ...` ❌) | 가능 (`p = ...` ✓) |
| 사용 | 거의 동일 | 거의 동일 |

> ⚠️ 시험 단골: `sizeof(a)` vs `sizeof(p)` 차이.
> 배열은 전체 크기, 포인터는 포인터 자체 크기(8바이트, 64bit 기준).

### 6.4 배열을 함수에 넘기면 포인터로 "decay"된다

```c
void sub(int b[], int n);    // 매개변수 b는 사실상 int* b
void sub(int *b, int n);     // 위와 완전히 동일!

int sum_indexed(int a[], int n) {       // 인덱스 표기법
    int sum = 0;
    for (int i = 0; i < n; i++) sum += a[i];
    return sum;
}

int sum_pointer(int *a, int n) {         // 포인터 표기법
    int sum = 0;
    int *p = a;
    for (int i = 0; i < n; i++) sum += *p++;
    return sum;
}
```

> 시험 출제 빈출: "왜 배열을 함수로 넘길 때 복사가 안 일어나는가?"
> → **배열 이름은 주소이고, 함수는 그 주소(포인터)만 받기 때문.**

### 🛠️ 학생 실습 #5 — 포인터로 배열 순회

> **문제:** `int arr[] = {5, 10, 15, 20, 25};` 배열을 **포인터만 사용해서** 모든 원소를 출력하시오. (인덱스 `[i]` 사용 금지)

<details>
<summary>👉 정답 코드</summary>

```c
#include <stdio.h>
#define SIZE 5

int main(void) {
    int arr[SIZE] = {5, 10, 15, 20, 25};
    int *p = arr;

    for (int i = 0; i < SIZE; i++) {
        printf("%d ", *p);    // 현재 가리키는 값 출력
        p++;                  // 다음 칸으로 이동
    }
    printf("\n");
    return 0;
}
```

**또는 한 줄로:**
```c
for (int *p = arr; p < arr + SIZE; p++)
    printf("%d ", *p);
```

</details>

## 7. 포인터를 쓰는 진짜 이유

시험 답변용으로 정리:

1. **참조에 의한 호출** — 함수가 외부 변수를 변경할 수 있다 (예: `swap`, `scanf`).
2. **여러 값을 반환** — `return`은 하나뿐이지만 포인터로는 N개 반환 가능.
3. **배열을 효율적으로 함수에 전달** — 복사 없이 주소만 넘김.
4. **동적 메모리 할당** — `malloc`, `free` (다음 단원 예고).
5. **연결 리스트, 트리 등 자료구조** — 노드끼리 주소로 연결.
6. **메모리 매핑 하드웨어 제어** — 임베디드(아두이노 등).

## 8. 시험 직전 체크리스트

| 질문 | 정답 |
|------|------|
| `&` 연산자의 역할? | 변수의 **주소**를 얻는다 |
| `*` 연산자의 역할? | 포인터가 가리키는 **값**을 얻는다 |
| `int *p1, p2;` → p2의 타입? | **int** (포인터 아님) |
| 초기화 안 한 포인터를 쓰면? | 쓰레기 주소 참조 → **크래시 위험** |
| 빈 포인터 표현 관용구? | `NULL` |
| `int *p`일 때 `p+1`은 몇 바이트 이동? | **4바이트 (int 크기)** |
| `*p++`과 `(*p)++`의 차이? | 전자는 **p를 이동**, 후자는 **값을 증가** |
| 함수가 외부 변수를 바꾸려면? | **참조에 의한 호출** (주소를 넘김) |
| `scanf`에서 `&`를 붙이는 이유? | scanf가 변수 주소를 알아야 그 자리에 값 저장 가능 |
| 함수가 받은 포인터로 값을 못 바꾸게 하려면? | 매개변수에 `const` |
| 지역변수 주소를 반환하면? | 함수 종료 후 사라진 변수 가리킴 → **위험** |
| 배열 이름 `a`는 무엇과 같은가? | `&a[0]` (첫 원소의 주소) |
| `a[i]`와 `*(a+i)`는? | **완전히 동일** |
| 함수에 배열을 넘기면 복사되는가? | 아니다, **주소만 전달**된다 |
| 배열을 함수 매개변수로 받으면? | 사실상 **포인터로 decay** |

---

## 9. 핵심 한 줄 요약

> **포인터 = "다른 변수의 주소를 담는 변수".**
> **`&`는 값→주소, `*`는 주소→값. 정확히 반대 연산.**
> **함수가 외부 변수를 바꾸려면 주소(`&`)를 넘기고, 안에서 `*`로 접근한다.**
> **배열 이름은 곧 첫 원소의 주소이며, `a[i]`와 `*(a+i)`는 같다.**
