# 실전 코딩 세션 2 — 도서 목록 관리 프로그램

> 단일 파일 / 사용 가능 개념: 변수·제어문·배열·함수·구조체·파일 I/O (텍스트)  
> 영화 꾸러미보다 한 단계씩 더 생각해야 한다.

---

## Step 1 — 구조체 정의 + 단일 데이터 출력 (15분)

### 과제

도서 1권의 정보를 구조체로 정의하고, 콘솔에 출력하라.

**요구사항**
- 필드: `title` (제목), `author` (저자), `year` (출판연도), `rating` (평점), `isRead` (읽음 여부, `int` 사용, 0 또는 1)
- `main()`에서 구조체 변수 1개를 직접 초기화
- 출력 형식:

```
제목: The Pragmatic Programmer
저자: David Thomas
출판연도: 1999
평점: 9.0
읽음: Yes
```

### 체크포인트

- [ ] 필드가 5개 모두 정의되어 있는가
- [ ] `isRead`를 `int`로 선언하고, 출력 시 `0`/`1`이 아닌 `Yes`/`No`로 변환하는가
- [ ] `%.1f`로 평점을 출력하는가
- [ ] 컴파일 및 실행 성공

## Step 2 — 배열 + 함수 구현 (20분)

### 과제

아래 함수 시그니처를 그대로 사용하여 구현하라.

```c
#define MAX 100

typedef struct {
    char title[100];
    char author[50];
    int year;
    float rating;
    int isRead;
} Book;

void printAll(Book books[], int count);
int addBook(Book books[], int count, Book newBook);
```

**요구사항**
- `printAll()` : 전체 목록 출력 (Step 1 형식 유지)
- `addBook()` : 배열이 가득 찼으면 `-1` 반환, 성공하면 `count + 1` 반환
- `main()`에서 3권을 직접 초기화한 뒤 `printAll()` 호출

### 체크포인트

- [ ] 함수 시그니처를 수정하지 않고 그대로 사용했는가
- [ ] `addBook()`의 반환값이 명세와 일치하는가
- [ ] `addBook()` 호출 후 반환값으로 `count`를 갱신하는가
- [ ] 컴파일 및 실행 성공

## Step 3 — 검색 (여러 건) + 정렬 (30분)

### 과제

저자 이름으로 검색하여 **여러 건**을 출력하고, 평점 내림차순으로 정렬하는 기능을 추가하라.

**요구사항**
- `searchByAuthor()` : 저자명이 일치하는 도서를 모두 출력, 없으면 `"검색 결과 없음"` 출력
- `sortByRating()` : 평점 내림차순 정렬 (버블 정렬 사용)

**출력 형식:**
```
[저자: David Thomas 검색 결과]
The Pragmatic Programmer | 1999 | 9.0 | Yes
```

### 체크포인트

- [ ] `searchByAuthor()`에서 일치하는 결과가 여러 건일 때 모두 출력하는가
- [ ] 검색 결과 건수를 카운트하여 0건일 때만 "검색 결과 없음"을 출력하는가
- [ ] `sortByRating()`에서 구조체 전체를 swap하는가 (포인터가 아닌 값 복사)
- [ ] 정렬 후 `printAll()`로 결과를 확인하는가
- [ ] 컴파일 및 실행 성공

## Step 4 — 파일 저장 / 불러오기 + 오류 처리 (25분)

### 과제

도서 목록을 텍스트 파일로 저장하고 불러오라. 오류 상황을 반드시 처리하라.

**요구사항**
- `saveToFile()` : `books.txt`에 저장
- `loadFromFile()` : `books.txt`에서 불러오기, 반환값으로 불러온 권수 반환
- 파일 형식:

```
The Pragmatic Programmer,David Thomas,1999,9.0,1
Clean Code,Robert Martin,2008,8.5,0
```

**오류 처리 요구사항**
- 파일 열기 실패 시 오류 메시지 출력 후 함수 종료
- `loadFromFile()` 에서 `MAX`를 초과하면 읽기 중단

### 체크포인트

- [ ] `fopen()` 반환값을 반드시 확인하는가
- [ ] `loadFromFile()`이 불러온 권수를 반환하는가
- [ ] `MAX` 초과 시 읽기를 중단하는가
- [ ] `fclose()`를 항상 호출하는가
- [ ] 파일이 없을 때 프로그램이 정상 동작하는가
- [ ] 컴파일 및 실행 성공


### 체크포인트
