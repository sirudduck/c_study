# C언어 프로그래밍 과제 — 콘솔 키오스크 프로그램

---

## 과제 개요

| 항목 | 내용 |
|------|------|
| 과제명 | 콘솔 키오스크 프로그램 |
| 제출 형식 | 여러 `.c` / `.h` 파일 (파일 분리 필수) |
| 배점 | 필수 기능 70점 + 도전 기능 30점 |
| 제출 방법 | 소스 파일 전체를 압축하여 제출 |

## 과제 배경

카페나 식당에서 흔히 볼 수 있는 **키오스크(Kiosk)** 를 콘솔 프로그램으로 구현한다.

실제 키오스크는 터치스크린과 카드 단말기가 연결되어 있지만,  
이 과제에서는 **키보드 입력과 콘솔 출력만으로** 동일한 흐름을 구현한다.

> 프로그램을 실행하면 아래와 같은 흐름으로 동작한다.
>
> ```
> 메뉴판 출력
>     ↓
> 손님이 메뉴 선택 (번호 입력)
>     ↓
> 장바구니에 담기 / 수량 변경 / 취소
>     ↓
> 주문 확인 및 가상 결제
>     ↓
> 주문 내역 파일 저장
>     ↓
> 다음 손님 또는 프로그램 종료
> ```

---

## 요구 사항 — 필수 기능 (70점)

### F-1. 파일 구조 분리 (10점)

제출하는 소스 코드는 반드시 아래 구조를 따른다.

```
kiosk/
├── main.c          ← main() 함수, 전체 흐름 제어
├── menu.h          ← MenuItem 구조체 정의, 메뉴 관련 함수 선언
├── menu.c          ← 메뉴 관련 함수 구현
├── cart.h          ← CartItem 구조체 정의, 장바구니 관련 함수 선언
├── cart.c          ← 장바구니 관련 함수 구현
├── order.h         ← Order 구조체 정의, 주문/결제 관련 함수 선언
├── order.c         ← 주문/결제 관련 함수 구현
└── orders.txt      ← 주문 내역이 저장되는 파일 (프로그램이 자동 생성)
```

**채점 기준:**
- 헤더 파일에 `#pragma once` 또는 `#ifndef` 가드가 있어야 한다.
- 함수 구현이 헤더 파일 안에 직접 작성되어 있으면 감점한다.
- `main.c`에 모든 코드를 몰아넣으면 파일 구조 점수 0점 처리한다.

---

### F-2. 구조체 정의 (10점)

아래 세 가지 구조체를 각 헤더 파일에 정의한다.

**`menu.h` — 메뉴 항목**

```c
typedef struct {
    int   id;           /* 메뉴 번호 (1부터 시작) */
    char  name[30];     /* 메뉴 이름 */
    int   price;        /* 가격 (원) */
    char  category[20]; /* 카테고리 (예: "음료", "음식", "디저트") */
} MenuItem;
```

**`cart.h` — 장바구니 항목**

```c
typedef struct {
    MenuItem item;  /* 담긴 메뉴 항목 */
    int      qty;   /* 수량 */
} CartItem;
```

**`order.h` — 주문 내역**

```c
typedef struct {
    int      order_id;          /* 주문 번호 */
    CartItem items[10];         /* 주문한 항목 (최대 10종류) */
    int      item_count;        /* 주문한 항목 종류 수 */
    int      total_price;       /* 총 결제 금액 */
    char     payment[10];       /* 결제 수단: "현금" 또는 "카드" */
} Order;
```

### F-3. 메뉴판 출력 (10점)

프로그램 시작 시 메뉴판을 출력한다.

**요구 사항:**
- 메뉴는 최소 9개 이상으로 구성하고, 카테고리별로 묶어서 출력한다.
- 메뉴 데이터는 `MenuItem` 배열로 관리하며, 하드코딩으로 초기화해도 된다.
- `menu.c` 안의 함수에서 출력을 담당해야 한다.

**출력 예시:**
```
========================================
           🍽  우리 식당 키오스크
========================================
[ 음식 ]
  1. 김치찌개        8,000 원
  2. 된장찌개        7,500 원
  3. 비빔밥          8,500 원

[ 음료 ]
  4. 콜라            2,000 원
  5. 사이다          2,000 원
  6. 오렌지주스      2,500 원

[ 디저트 ]
  7. 아이스크림      2,500 원
  8. 케이크 한 조각  4,000 원
  9. 마카롱          1,500 원
========================================
```

### F-4. 장바구니 기능 (15점)

손님이 메뉴를 선택하고 장바구니를 관리할 수 있어야 한다.

**구현해야 할 함수 (cart.c):**

```c
/* 장바구니에 메뉴 추가. 이미 있으면 수량만 증가 */
void cart_add(CartItem cart[], int *count, MenuItem item, int qty);

/* 장바구니에서 특정 메뉴 제거 */
void cart_remove(CartItem cart[], int *count, int menu_id);

/* 장바구니 전체 출력 */
void cart_print(CartItem cart[], int count);

/* 장바구니 총 금액 계산 */
int  cart_total(CartItem cart[], int count);

/* 장바구니 비우기 */
void cart_clear(CartItem cart[], int *count);
```

**동작 예시:**
```
[ 장바구니 ]
  1. 김치찌개   x 2   = 16,000 원
  2. 콜라       x 1   =  2,000 원
─────────────────────────────────
  합계                  18,000 원

0. 주문 취소    8. 항목 삭제    9. 결제로 이동
```

**채점 기준:**
- 같은 메뉴를 두 번 추가하면 수량이 증가해야 한다.
- 장바구니가 비어있을 때 결제로 이동하려 하면 경고 메시지를 출력해야 한다.
- 장바구니 최대 수용 항목 종류: 10종류.

---

### F-5. 가상 결제 기능 (10점)

결제 수단을 선택하고 금액을 처리한다.

**현금 결제 흐름:**
```
총 금액: 18,000 원
결제 수단을 선택하세요.
1. 현금   2. 카드
> 1

투입 금액을 입력하세요: 20000
거스름돈: 2,000 원
결제 완료!
```

**카드 결제 흐름:**
```
총 금액: 18,000 원
결제 수단을 선택하세요.
1. 현금   2. 카드
> 2

카드를 단말기에 꽂아주세요... (엔터를 누르면 승인)
결제 완료!
```

**채점 기준:**
- 현금 투입 금액이 총 금액보다 적으면 부족하다는 메시지를 출력하고 다시 입력받아야 한다.
- 거스름돈이 0원이면 거스름돈 출력을 생략해도 된다.

---

### F-6. 주문 내역 파일 저장 (15점)

결제 완료 시 주문 내역을 `orders.txt` 파일에 **이어쓰기(append)** 로 저장한다.

**저장 형식 (자유롭게 구성해도 되나 아래 정보는 반드시 포함):**
```
========================================
주문번호: 1
----------------------------------------
김치찌개       x2    16000원
콜라           x1     2000원
----------------------------------------
합계: 18000원
결제수단: 현금
========================================
```

**요구 사항:**
- 파일이 없으면 새로 생성, 있으면 기존 내용 뒤에 이어쓰기.
- 주문 번호는 프로그램 실행 중 자동으로 1씩 증가한다.
- 파일 저장 실패 시 오류 메시지를 출력해야 한다.

**채점 기준:**
- `fopen` / `fprintf` / `fclose` 를 올바르게 사용해야 한다.
- 이어쓰기 모드(`"a"`)를 사용해야 한다.

## 요구 사항 — 도전 기능 (30점)

> 도전 기능은 선택 사항이다. 구현하지 않아도 필수 기능 점수에 영향을 주지 않는다.  
> 단, 도전 기능을 구현하다가 필수 기능이 망가지면 감점한다.  
> 안전하게 별도 함수로 분리해서 구현할 것.

---

### C-1. 메뉴 파일에서 불러오기 (10점)

하드코딩된 메뉴 대신, `menu.txt` 파일에서 메뉴 데이터를 읽어온다.

**`menu.txt` 형식:**
```
1,김치찌개,8000,음식
2,된장찌개,7500,음식
3,비빔밥,8500,음식
4,콜라,2000,음료
5,사이다,2000,음료
6,오렌지주스,2500,음료
7,아이스크림,2500,디저트
8,케이크 한 조각,4000,디저트
9,마카롱,1500,디저트
```

**요구 사항:**
- `fscanf` 또는 `fgets` + 파싱으로 읽어온다.
- 파일이 없으면 하드코딩된 기본 메뉴를 사용한다.

### C-2. 주문 내역 조회 기능 (10점)

저장된 `orders.txt` 파일을 읽어서 화면에 출력하는 기능을 추가한다.

**메인 메뉴에 항목 추가:**
```
========================================
  1. 주문하기
  2. 지난 주문 내역 보기    ← 이걸 추가
  0. 종료
========================================
```

**요구 사항:**
- `orders.txt`가 없으면 "저장된 주문 내역이 없습니다."를 출력한다.
- 파일 전체 내용을 그대로 출력해도 된다.

---

### C-3. 메뉴 정렬 및 검색 (10점)

아래 두 가지 기능 중 하나 이상을 구현한다. 둘 다 구현하면 10점 전부 받는다.

**정렬 (5점):**
- 가격 오름차순 또는 내림차순으로 메뉴를 정렬하여 출력하는 기능.
- 정렬 알고리즘은 버블 정렬, 선택 정렬 등 자유롭게 선택.

**검색 (5점):**
- 메뉴 이름의 일부를 입력하면 해당 메뉴를 찾아 출력하는 기능.
- 예: "찌개" 입력 시 김치찌개, 된장찌개 모두 출력.
- `strstr()` 함수 사용 가능 (`string.h`).

## 힌트 및 허용 사항

**허용:**
- 메뉴 데이터 하드코딩 초기화 허용 (C-1 미구현 시).
- 카테고리, 메뉴 이름, 가격은 자유롭게 변경 가능.
- 콘솔 화면 정리(`system("cls")` 또는 `system("clear")`)는 선택 사항.

**힌트:**
- 장바구니에서 "이미 있는 메뉴인지 확인"은 `id` 비교로 구현하면 된다.
- 파일 이어쓰기는 `fopen("orders.txt", "a")` 모드를 사용한다.
- `strstr(haystack, needle)` 은 문자열 안에서 부분 문자열을 찾아준다.
- 주문 번호를 전역 변수로 관리할 경우 `extern` 을 활용할 수 있다.
- `fprintf(fp, "%s", str)` 으로 파일에 문자열을 쓸 수 있다.

**금지:**
- 동적 메모리 할당(`malloc`, `calloc`, `realloc`, `free`) 사용 금지.
- 전역 배열 크기: 메뉴 최대 20개, 장바구니 최대 10종류, 주문 내역 최대 50건.
