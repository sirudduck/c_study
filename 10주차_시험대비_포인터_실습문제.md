## Part C. 코드 작성 문제

### C-1 ⭐ — 두 값 교환
포인터를 사용하여 두 정수를 교환하는 `swap` 함수를 직접 작성하고, main에서 호출해 결과를 출력하시오.

<details>
<summary>👉 정답 예시</summary>

```c
#include <stdio.h>

void swap(int *px, int *py) {
    int tmp = *px;
    *px = *py;
    *py = tmp;
}

int main(void) {
    int a = 1, b = 2;
    printf("교환 전: a=%d, b=%d\n", a, b);
    swap(&a, &b);
    printf("교환 후: a=%d, b=%d\n", a, b);
    return 0;
}
```

</details>

### C-2 ⭐⭐ — 절대값 함수
`int` 변수의 절대값을 **포인터를 통해** 변경하는 함수 `make_abs(int *p)`를 작성하시오.
- 양수이면 그대로, 음수이면 양수로 바꿈.

<details>
<summary>👉 정답 예시</summary>

```c
#include <stdio.h>

void make_abs(int *p) {
    if (*p < 0) *p = -(*p);
}

int main(void) {
    int x = -42, y = 17;
    make_abs(&x);
    make_abs(&y);
    printf("x = %d, y = %d\n", x, y);     // x = 42, y = 17
    return 0;
}
```

</details>

### C-3 ⭐⭐ — 통계 함수 (다중 반환)
정수 배열을 받아 **합, 평균, 최대값, 최소값**을 한 번에 계산하는 함수를 작성하시오.
```c
void get_stats(const int a[], int n,
               int *sum, double *avg, int *min, int *max);
```

<details>
<summary>👉 정답 예시</summary>

```c
#include <stdio.h>
#define SIZE 6

void get_stats(const int a[], int n,
               int *sum, double *avg, int *min, int *max) {
    *sum = 0;
    *min = *max = a[0];
    for (int i = 0; i < n; i++) {
        *sum += a[i];
        if (a[i] < *min) *min = a[i];
        if (a[i] > *max) *max = a[i];
    }
    *avg = (double)*sum / n;
}

int main(void) {
    int arr[SIZE] = {15, 32, 7, 88, 24, 56};
    int s, mn, mx;
    double a;
    get_stats(arr, SIZE, &s, &a, &mn, &mx);
    printf("합=%d, 평균=%.2f, 최소=%d, 최대=%d\n", s, a, mn, mx);
    return 0;
}
```

**예상 출력:** `합=222, 평균=37.00, 최소=7, 최대=88`

</details>

### C-4 ⭐⭐ — 배열 뒤집기 (in-place)
포인터를 두 개 사용하여 배열을 **그 자리에서** 뒤집는 함수를 작성하시오.
```c
void reverse(int a[], int n);
```

<details>
<summary>👉 정답 예시</summary>

```c
#include <stdio.h>
#define SIZE 7

void reverse(int a[], int n) {
    int *left  = a;
    int *right = a + n - 1;
    while (left < right) {
        int tmp = *left;
        *left   = *right;
        *right  = tmp;
        left++;
        right--;
    }
}

int main(void) {
    int arr[SIZE] = {1, 2, 3, 4, 5, 6, 7};
    reverse(arr, SIZE);
    for (int i = 0; i < SIZE; i++) printf("%d ", arr[i]);
    printf("\n");
    return 0;
}
```

**예상 출력:** `7 6 5 4 3 2 1`

**시험 포인트:** 인덱스 대신 두 포인터(`left`, `right`)로 양 끝에서 안쪽으로 좁혀가는 패턴. 이 기법은 시험·실무에서 모두 자주 쓰인다.

</details>

### C-5 ⭐⭐⭐ — 포인터로 문자열 길이 (strlen 직접 구현)
`while`문과 포인터만 사용해 문자열의 길이를 구하는 함수를 작성하시오.
```c
int my_strlen(const char *s);
```

> 힌트: C 문자열은 `'\0'`(null 문자)으로 끝난다. 포인터를 따라가다 `'\0'`을 만나면 멈춘다.

<details>
<summary>👉 정답 예시</summary>

```c
#include <stdio.h>

int my_strlen(const char *s) {
    const char *p = s;
    while (*p != '\0') p++;
    return p - s;                 // 포인터끼리 뺄셈 = 원소 개수
}

int main(void) {
    printf("%d\n", my_strlen("Hello"));         // 5
    printf("%d\n", my_strlen("프로그래밍"));     // 한글은 UTF-8 기준 바이트 수
    return 0;
}
```

**시험 포인트:** 같은 배열 안에서는 포인터끼리 뺄셈이 가능하고, **그 결과는 원소 개수**다.

</details>

### C-6 ⭐⭐⭐ — 자율주행 센서 시뮬 (PDF 미니 프로젝트)
`getSensorData(double *p)` 함수를 작성하여 0~99 사이 난수 3개를 배열에 채워 넣고, main에서 결과를 출력하시오.

<details>
<summary>👉 정답 예시</summary>

```c
#include <stdio.h>
#include <stdlib.h>
#include <time.h>

void getSensorData(double *p) {
    for (int i = 0; i < 3; i++) {
        *p = rand() % 100;        // 0~99 난수
        p++;                      // 다음 칸으로
    }
}

int main(void) {
    srand((unsigned)time(NULL));
    double sensorData[3];
    getSensorData(sensorData);

    printf("왼쪽   센서: %.2f\n", sensorData[0]);
    printf("중간   센서: %.2f\n", sensorData[1]);
    printf("오른쪽 센서: %.2f\n", sensorData[2]);
    return 0;
}
```

**시험 포인트:** 배열 이름을 `double *p`로 받으면 함수 안에서 포인터 연산(`*p`, `p++`)이 자유롭게 가능. 배열은 이미 호출자의 원본이므로 함수 내 수정이 그대로 반영된다.

</details>

## Part D. 종합 도전 문제 ⭐⭐⭐

### D-1 — 배열 두 개 합치기
정렬된 두 배열 `a`, `b`를 받아 **하나의 정렬된 배열 `c`**로 합치는 함수를 작성하시오.
```c
void merge(const int *a, int na, const int *b, int nb, int *c);
```

예: `a = {1,3,5,7}`, `b = {2,4,6,8}` → `c = {1,2,3,4,5,6,7,8}`

<details>
<summary>👉 정답 예시</summary>

```c
#include <stdio.h>

void merge(const int *a, int na, const int *b, int nb, int *c) {
    int i = 0, j = 0, k = 0;
    while (i < na && j < nb) {
        if (a[i] <= b[j]) c[k++] = a[i++];
        else              c[k++] = b[j++];
    }
    while (i < na) c[k++] = a[i++];     // a에 남은 것
    while (j < nb) c[k++] = b[j++];     // b에 남은 것
}

int main(void) {
    int a[] = {1, 3, 5, 7};
    int b[] = {2, 4, 6, 8};
    int c[8];
    merge(a, 4, b, 4, c);
    for (int i = 0; i < 8; i++) printf("%d ", c[i]);
    return 0;
}
```

**예상 출력:** `1 2 3 4 5 6 7 8`

</details>

## 시험 직전 흔한 실수 Top 7

1. **`int *p1, p2;` → p2를 포인터로 착각** — `*`는 변수마다 붙여야 한다.
2. **초기화 없이 `*p` 사용** — 반드시 유효한 주소를 대입한 뒤에 역참조.
3. **`scanf`에서 `&` 누락** — `scanf("%d", n)` ❌ → `scanf("%d", &n)` ✓
4. **함수에서 값을 바꿀 때 매개변수를 그냥 변수로 받음** — 주소(`*`)로 받아야 원본 변경 가능.
5. **`p + 1`이 1바이트 이동한다고 착각** — 자료형 크기만큼 이동.
6. **`*p++`과 `(*p)++` 혼동** — 괄호 위치가 의미를 완전히 바꿈.
7. **지역변수 주소 반환** — 함수 종료 후 메모리 회수됨.

## 시험 1시간 전 마지막 외울 것

```
[연산자]
&x   → x의 주소
*p   → p가 가리키는 값

[배열 ↔ 포인터]
a == &a[0]
a[i] == *(a+i) == p[i] == *(p+i)

[함수 인수 전달]
값 호출 : 복사본    → 원본 안 바뀜
참조 호출: 주소     → 원본 바뀜  (스왑, scanf 패턴)

[NULL]
int *p = NULL;     // 안전한 초기화 관용구

[const]
void f(const int *p)   // p로 값 못 바꿈을 약속

[포인터 연산]
p + n  →  n * sizeof(*p) 바이트 이동
```

이 박스만 머리에 박혀 있어도 시험에서 50% 이상은 잡힌다.
