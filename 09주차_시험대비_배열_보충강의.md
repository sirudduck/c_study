## 0. 들어가기 — 왜 배열이 필요한가?

### 0.1 변수 5개 vs 접시 한 줄

지금까지 배운 변수는 이렇게 생겼다.

```c
int score1 = 80;
int score2 = 70;
int score3 = 90;
int score4 = 60;
int score5 = 100;
```

학생 5명일 땐 괜찮지만, 30명이라면? 100명이라면?

> 🍳 **주방 비유**
> 변수 하나는 **반찬통 한 개**다.
> 학생 100명의 점수를 저장하려면 반찬통 100개를 따로따로 꺼내야 한다.
> 그리고 각 반찬통의 이름(`score1`, `score2`, ...)을 전부 외워야 한다.
>
> **배열은 칸이 나뉜 한 줄짜리 긴 접시다.**
> 접시 하나(`scores`)만 꺼내면, 칸 번호(`scores[0]`, `scores[1]`, ...)로 모두 접근할 수 있다.

### 0.2 배열의 핵심 3가지 (시험 단골)

1. **메모리에 연속적으로 저장된다.** (조리대 위에 칸이 붙어 있는 한 줄짜리 접시)
2. **하나의 이름을 공유하고 번호(인덱스)만 다르다.**
3. **인덱스는 0부터 시작한다.** (n개짜리 배열의 마지막 번호는 `n-1`)

## 1. 배열의 선언과 기본 사용

### 1.1 선언 문법

```c
자료형 배열이름[원소개수];

int   score[60];   // int형 60개짜리 접시
float cost[12];    // float형 12개짜리 접시
char  name[50];    // char형 50개짜리 접시
```

### 1.2 원소 접근

```c
scores[5] = 80;            // 6번째 칸에 80을 담는다
scores[1] = scores[0];     // 1번 칸의 값을 2번 칸에 복사
scores[i] = 100;           // 변수도 인덱스로 가능
scores[i + 2] = 100;       // 수식도 인덱스로 가능
```

> 🍳 **주방 비유**
> `scores[i]`는 "접시의 i번 칸"이다.
> 칸 번호만 알면 어떤 음식이든 그 자리에 담을 수 있다.

### 1.3 시험에 자주 나오는 함정 — **인덱스 범위 초과**

```c
int scores[5];     // 칸은 0, 1, 2, 3, 4 → 5칸
scores[5] = 60;    // 치명적 오류! 6번째 칸은 존재하지 않는다
```

> ⚠️ C 컴파일러는 **에러를 내지 않는다.**
> 실행은 되지만, 옆 메모리(다른 변수의 영역)를 침범한다.
> → 프로그래머가 책임지고 범위를 지켜야 한다.

### 🛠️ 학생 실습 #1 — 기본 출력

다음을 작성해 보자.

> **문제:** 크기 5인 `int` 배열에 10, 20, 30, 40, 50을 차례로 넣고, `for`문으로 출력하시오.

<details>
<summary>👉 정답 코드 (먼저 직접 해보고 펼치자)</summary>

```c
#include <stdio.h>
#define SIZE 5

int main(void) {
    int scores[SIZE];

    for (int i = 0; i < SIZE; i++)
        scores[i] = (i + 1) * 10;       // 10, 20, 30, 40, 50

    for (int i = 0; i < SIZE; i++)
        printf("scores[%d] = %d\n", i, scores[i]);

    return 0;
}
```

</details>

## 2. 배열의 초기화 (★ 출제 빈도 매우 높음)

### 2.1 4가지 초기화 패턴

```c
// 패턴 ①: 완전 초기화
int a[5] = {31, 63, 62, 87, 14};

// 패턴 ②: 부분 초기화 → 나머지는 자동으로 0
int b[5] = {31, 63};                 // → {31, 63, 0, 0, 0}

// 패턴 ③: 크기 생략 → 컴파일러가 자동 계산
int c[] = {1, 2, 3, 4, 5, 6};        // 크기 6으로 자동 결정

// 패턴 ④: 초기화 안 함 → 쓰레기값 (지역변수의 경우)
int d[5];                            // 내용 예측 불가!
```

### 2.2 시험 함정 정리

| 코드 | 결과 |
|------|------|
| `int a[5] = {0};` | **전체가 0으로 초기화** (자주 쓰는 관용구) |
| `int a[5] = {1};` | `{1, 0, 0, 0, 0}` — 첫 칸만 1, 나머지 0 |
| `int a[5];`<br>(지역변수) | 쓰레기값 (값 예측 불가) |
| `int a[5] = {1, 2, 3, 4, 5, 6};` | **컴파일 에러** (초기값이 더 많음) |

### 2.3 배열 크기 자동 계산

```c
int scores[] = {1, 2, 3, 4, 5, 6};
int size = sizeof(scores) / sizeof(scores[0]);   // = 6
```

> 🍳 **왜 `sizeof(scores) / sizeof(scores[0])`인가?**
> `sizeof(scores)`는 접시 전체의 무게(바이트).
> `sizeof(scores[0])`는 칸 하나의 무게.
> 전체 무게를 칸 하나의 무게로 나누면 **칸 개수**가 나온다.

### 2.4 시험에 자주 나오는 두 가지 금지사항

```c
int a[5] = {1, 2, 3, 4, 5};
int b[5];

// ❌ 금지 ①: 배열 통째로 대입 불가
b = a;                  // 컴파일 에러!

// ❌ 금지 ②: 배열 통째로 비교 불가
if (a == b) { ... }     // 컴파일은 되지만 의미가 다름 (주소 비교)

// ✅ 올바른 방법: 원소를 하나씩
for (int i = 0; i < 5; i++) b[i] = a[i];   // 복사
for (int i = 0; i < 5; i++)
    if (a[i] != b[i]) { /* 다름 */ }
```

## 3. 배열과 함수 (★★ 시험 핵심)

### 3.1 가장 중요한 한 문장

> **배열을 함수에 전달하면 "주소"가 전달된다. 사본이 아니라 원본이다.**

> 🍳 **주방 비유**
> 일반 변수는 **반찬통의 내용물을 복사해서** 다른 요리사에게 넘기는 것.
> 배열은 **접시 그 자체를 통째로 넘기는 것**.
> → 받은 요리사가 칸의 내용을 바꾸면, 원본 접시의 내용이 바뀐다.

### 3.2 예제 — 배열을 함수로 전달

```c
#include <stdio.h>
#define SIZE 7

void modify_array(int a[], int size) {
    for (int i = 0; i < size; i++)
        ++a[i];                       // 원본이 바뀐다!
}

void print_array(int a[], int size) {
    for (int i = 0; i < size; i++)
        printf("%3d ", a[i]);
    printf("\n");
}

int main(void) {
    int list[SIZE] = {1, 2, 3, 4, 5, 6, 7};

    print_array(list, SIZE);          // 1 2 3 4 5 6 7
    modify_array(list, SIZE);
    print_array(list, SIZE);          // 2 3 4 5 6 7 8

    return 0;
}
```

### 3.3 원본 보호 — `const` 키워드 (출제 빈출)

```c
void print_array(const int a[], int size) {
    // a[0] = 100;   // ❌ 컴파일 에러 — const가 변경을 막는다
    for (int i = 0; i < size; i++)
        printf("%d ", a[i]);
}
```

> ⚠️ **시험 포인트:** 함수가 받은 배열을 절대 변경하지 않을 것임을 약속할 때 `const`를 붙인다.

### 🛠️ 학생 실습 #3 — 평균 함수

> **문제:** `int` 배열과 크기를 받아 평균(double)을 반환하는 함수 `get_average`를 작성하라.
>
> 예: `{10, 20, 30, 40, 50}` → 30.0

<details>
<summary>👉 정답 코드</summary>

```c
#include <stdio.h>
#define SIZE 5

double get_average(const int a[], int n) {
    int sum = 0;
    for (int i = 0; i < n; i++) sum += a[i];
    return (double)sum / n;
}

int main(void) {
    int scores[SIZE] = {10, 20, 30, 40, 50};
    printf("평균 = %.2f\n", get_average(scores, SIZE));
    return 0;
}
```

**시험 포인트:** `(double)sum / n`처럼 캐스팅을 해야 소수점이 나온다. `sum / n`만 쓰면 정수 나눗셈으로 30이 된다.

</details>

## 4. 자주 출제되는 알고리즘 3종

### 4.1 최소값 / 최대값 찾기

> **레시피:** 첫 번째 원소를 일단 "최소"라고 가정한 뒤, 나머지를 차례로 비교.

```c
int find_min(const int a[], int n) {
    int minimum = a[0];                       // 일단 첫 칸을 최소로 가정
    for (int i = 1; i < n; i++)
        if (a[i] < minimum)
            minimum = a[i];                   // 더 작은 값 발견 시 갱신
    return minimum;
}
```

**암기 포인트:** `i = 1`부터 시작 (0번은 이미 가정값으로 썼으므로).

### 4.2 선택 정렬 (Selection Sort) — 단골 출제

> **레시피:**
> 1. 정렬 안 된 부분에서 **최소값의 위치**를 찾는다.
> 2. 그 자리를 **현재 자리**와 교환한다.
> 3. 다음 자리로 넘어가서 반복.

```c
#include <stdio.h>
#define SIZE 10

int main(void) {
    int list[SIZE] = {3, 2, 9, 7, 1, 4, 8, 0, 6, 5};
    int i, j, temp, least;

    for (i = 0; i < SIZE - 1; i++) {
        least = i;                            // (i+1)번부터 끝까지 중 최소 인덱스 찾기
        for (j = i + 1; j < SIZE; j++)
            if (list[j] < list[least])
                least = j;

        // 교환 (★ 반드시 temp 사용)
        temp = list[i];
        list[i] = list[least];
        list[least] = temp;
    }

    for (i = 0; i < SIZE; i++) printf("%d ", list[i]);
    return 0;
}
```

> ⚠️ **시험 단골 함정 — 값 교환은 반드시 임시 변수로!**
>
> ```c
> // ❌ 잘못 — 첫 줄에서 list[i]의 원래 값이 사라진다
> list[i]     = list[least];
> list[least] = list[i];
>
> // ✅ 올바름
> temp        = list[i];
> list[i]     = list[least];
> list[least] = temp;
> ```

### 4.3 탐색 — 순차 vs 이진

| 구분 | 순차 탐색 | 이진 탐색 |
|------|-----------|-----------|
| 전제 조건 | 없음 | **정렬되어 있어야 함** |
| 방식 | 0번부터 차례로 비교 | 중앙값과 비교 후 절반 버리기 |
| 시간복잡도 | O(n) | O(log n) |

**순차 탐색 (간단):**
```c
for (int i = 0; i < SIZE; i++)
    if (list[i] == key)
        printf("탐색 성공 인덱스 = %d\n", i);
```

**이진 탐색 (★ 시험 단골):**
```c
int binary_search(int list[], int n, int key) {
    int low = 0, high = n - 1, middle;
    while (low <= high) {
        middle = (low + high) / 2;
        if (key == list[middle]) return middle;     // 찾음
        else if (key > list[middle]) low = middle + 1;
        else                          high = middle - 1;
    }
    return -1;                                       // 못 찾음
}
```

> 🍳 **이진 탐색 비유 — 사전 찾기**
> 두꺼운 영어 사전에서 "kitchen"을 찾을 때, 첫 페이지부터 넘기는 사람은 없다.
> 책 한가운데를 열고, k가 거기보다 뒤니까 뒷쪽 절반에서 또 가운데를 연다.
> → **매번 후보가 반씩 줄어드는 것**이 이진 탐색의 핵심.

### 🛠️ 학생 실습 #4 — 최대값과 그 위치

> **문제:** 배열에서 최대값과 그 인덱스를 동시에 출력하는 프로그램을 작성하라.
>
> 입력: `{28, 81, 60, 83, 67, 10, 66, 97, 37, 94}`
> 출력: `최대값 = 97, 인덱스 = 7`

<details>
<summary>👉 정답 코드</summary>

```c
#include <stdio.h>
#define SIZE 10

int main(void) {
    int prices[SIZE] = {28, 81, 60, 83, 67, 10, 66, 97, 37, 94};
    int max_val = prices[0];
    int max_idx = 0;

    for (int i = 1; i < SIZE; i++) {
        if (prices[i] > max_val) {
            max_val = prices[i];
            max_idx = i;
        }
    }
    printf("최대값 = %d, 인덱스 = %d\n", max_val, max_idx);
    return 0;
}
```

**시험 포인트:** 위치(인덱스)도 같이 갱신해야 한다. `max_val`만 갱신하면 위치 정보를 잃는다.

</details>

## 5. 2차원 배열 (★ 시험 핵심)

### 5.1 선언과 의미

```c
int s[3][5];   // 3행 5열 → 총 15칸
```

> 🍳 **주방 비유**
> 1차원 배열이 **한 줄짜리 긴 접시**라면,
> 2차원 배열은 **3단 도시락 (한 단에 5칸씩)**.

### 5.2 메모리 배치 — **실제로는 한 줄로 깔린다**

```c
int s[3][5] = {
    {0,  1,  2,  3,  4},
    {10, 11, 12, 13, 14},
    {20, 21, 22, 23, 24}
};
```

메모리 안에서는 이렇게 일렬로 놓인다:

```
[ 0][ 1][ 2][ 3][ 4][10][11][12][13][14][20][21][22][23][24]
└── 0번 행 ──┘└──── 1번 행 ────┘└──── 2번 행 ────┘
```

**핵심:** 사람 눈에는 2차원이지만 메모리는 1차원 직선. **행 우선(row-major) 저장.**

### 5.3 초기화 — 3가지 방식

```c
// 방식 ① 행마다 중괄호로 구분 (가독성 최고)
int s[3][5] = {
    {0,  1,  2,  3,  4},
    {10, 11, 12, 13, 14},
    {20, 21, 22, 23, 24}
};

// 방식 ② 행 개수 생략 가능 (열은 반드시 명시)
int s[][5] = {
    {0,  1,  2,  3,  4},
    {10, 11, 12, 13, 14}
};

// 방식 ③ 한 줄로 (실수 위험 ↑)
int s[][5] = {0, 1, 2, 3, 4, 10, 11, 12, 13, 14, 20, 21, 22, 23, 24};
```

> ⚠️ **시험 포인트:** 행은 생략 가능하지만 **열은 절대 생략 못 한다.**
> 컴파일러가 "한 행의 길이"를 알아야 메모리 주소를 계산할 수 있기 때문.

### 5.4 2차원 배열을 함수로 전달

```c
#define ROWS 3
#define COLS 5

int sum(int scores[ROWS][COLS]) {     // 또는 int scores[][COLS]
    int total = 0;
    for (int r = 0; r < ROWS; r++)
        for (int c = 0; c < COLS; c++)
            total += scores[r][c];
    return total;
}
```

**시험 단골:** 함수 매개변수에서 첫 번째 차원(`ROWS`)은 생략 가능하지만, **두 번째 차원(`COLS`)은 반드시 명시**해야 한다.

### 🛠️ 학생 실습 #5 — 행렬 덧셈

> **문제:** 3×3 행렬 A와 B를 더하여 C에 저장하고 출력하시오.
> A = `{{2,3,0},{8,9,1},{7,0,5}}`, B = `{{1,0,0},{1,0,0},{1,0,0}}`

<details>
<summary>👉 정답 코드</summary>

```c
#include <stdio.h>
#define ROWS 3
#define COLS 3

int main(void) {
    int A[ROWS][COLS] = {{2,3,0}, {8,9,1}, {7,0,5}};
    int B[ROWS][COLS] = {{1,0,0}, {1,0,0}, {1,0,0}};
    int C[ROWS][COLS];

    for (int r = 0; r < ROWS; r++)
        for (int c = 0; c < COLS; c++)
            C[r][c] = A[r][c] + B[r][c];

    for (int r = 0; r < ROWS; r++) {
        for (int c = 0; c < COLS; c++)
            printf("%d ", C[r][c]);
        printf("\n");
    }
    return 0;
}
```

**예상 출력:**
```
3 3 0
9 9 1
8 0 5
```

</details>

## 6. 시험 직전 체크리스트

다음 질문에 즉답할 수 있어야 한다. (답을 가린 후 스스로 답해 보자)

1. `int a[10];`에서 첫 번째 원소의 인덱스는? → **0**
2. `int a[10];`에서 마지막 원소의 인덱스는? → **9**
3. `int a[5] = {1};`의 결과는? → **{1, 0, 0, 0, 0}**
4. `int a[] = {1, 2, 3};`에서 a의 크기는? → **3**
5. 배열 두 개를 `if (a == b)`로 비교할 수 있는가? → **아니다 (주소 비교가 됨)**
6. 배열 두 개를 `a = b;`로 대입할 수 있는가? → **아니다 (컴파일 에러)**
7. 배열을 함수에 넘기면 원본인가 사본인가? → **원본 (주소가 전달)**
8. 함수가 배열을 변경하지 못하게 막는 키워드? → **`const`**
9. 선택 정렬에서 두 값을 교환할 때 임시 변수가 필요한 이유? → **첫 줄에서 원래 값이 덮어쓰여 사라지기 때문**
10. `int a[3][5];`를 함수 매개변수로 받을 때 반드시 명시해야 하는 차원은? → **두 번째 차원(열 크기)**
11. 이진 탐색의 전제 조건은? → **배열이 정렬되어 있을 것**
12. `sizeof(arr) / sizeof(arr[0])`이 의미하는 것은? → **배열 원소 개수**

## 7. 핵심 한 줄 요약

> **배열 = 같은 이름을 공유하는 칸 나뉜 한 줄짜리 접시.**
> **인덱스는 0부터, 마지막은 크기-1, 통째 대입·비교 금지, 함수로 넘기면 원본 변경 가능.**
> **2차원 배열도 메모리에는 한 줄로 깔리고(행 우선), 함수 매개변수에서 열 크기는 절대 생략 못 한다.**
