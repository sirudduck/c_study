# 실습 2 — RGB 픽셀 보정기 (응용)

> **난이도**: ⭐⭐⭐☆☆ (보통)
> **소요 시간**: 약 40분
> **학습 목표**: `const` 의 위치별 의미를 코드로 체득한다. 함수 매개변수의 `const` 관례를 익힌다.

---

## 🎯 시나리오

여러분은 카메라 영상 처리 모듈을 개발하고 있습니다. 픽셀 데이터(RGB) 를 다루는 함수들을 만들어야 하는데, 이 함수들은 각각 **다른 권한**을 가져야 합니다.

- 어떤 함수는 픽셀을 **읽기만** 해야 합니다 (분석용)
- 어떤 함수는 픽셀의 **값을 바꿔야** 합니다 (필터 적용)
- 어떤 함수는 **여러 픽셀 중 어느 것을 가리킬지** 결정해야 합니다 (선택)

이런 권한을 컴파일러가 **강제**해주면 어떨까요? `const` 가 바로 그 역할을 합니다.

---

## 🎨 RGB 픽셀 구조

```c
typedef struct {
    unsigned char r;   // 빨강 (0~255)
    unsigned char g;   // 초록 (0~255)
    unsigned char b;   // 파랑 (0~255)
} Pixel;
```

---

## 📋 요구사항

다음 4개의 함수를 완성하세요. **각 함수의 매개변수에 `const` 를 어디에 붙일지** 가 핵심입니다.

### 1. `print_pixel` — 픽셀 정보 출력 (읽기 전용)
이 함수는 픽셀을 **절대 수정하지 않음**을 컴파일러가 강제하도록 해야 합니다.

### 2. `brighten_pixel` — 픽셀 밝기 증가 (값 수정)
RGB 각 값에 일정량을 더합니다. 단, 255를 넘으면 255로 클램핑 (clamping) 합니다.
이 함수는 **포인터로 받은 픽셀의 값**을 바꿔야 합니다.

### 3. `invert_pixel` — 색상 반전 (값 수정)
각 채널을 `255 - 값` 으로 바꿉니다 (네거티브 효과).

### 4. `find_brightest` — 가장 밝은 픽셀의 주소 반환
픽셀 배열에서 R+G+B 합이 가장 큰 픽셀을 찾아, **그 픽셀의 주소를 반환**합니다.
함수는 배열을 **수정하지 않으며**, 반환된 포인터를 통해서도 **수정할 수 없게** 해야 합니다.

## 🧩 코드 골격 — 빈칸을 채우세요

```c
#include <stdio.h>

typedef struct {
    unsigned char r;
    unsigned char g;
    unsigned char b;
} Pixel;

/* ─────────────────────────────────────────────
 * [함수 1] 픽셀 정보 출력 — 읽기 전용
 * ───────────────────────────────────────────── */
void print_pixel(/* 빈칸 1
                    힌트: 픽셀을 가리키는 포인터를 받되,
                          그 픽셀의 내용을 수정 못 하게 막음 */)
{
    printf("RGB(%3d, %3d, %3d)\n", p->r, p->g, p->b);

    /* 아래 줄의 주석을 풀면 컴파일 에러가 나야 정상! */
    /* p->r = 0; */
}

/* ─────────────────────────────────────────────
 * [함수 2] 픽셀 밝기 증가 — 값 수정 가능
 * ───────────────────────────────────────────── */
void brighten_pixel(/* 빈칸 2
                       힌트: 픽셀을 가리키는 포인터를 받고,
                             그 내용을 수정해야 함 */,
                    int amount)
{
    /* 빈칸 3: r 채널을 amount 만큼 증가시키되 255를 넘지 않도록
       힌트: (값 + amount > 255) ? 255 : (값 + amount) */
    int new_r = ____________________;
    int new_g = ____________________;
    int new_b = ____________________;

    p->r = (unsigned char)new_r;
    p->g = (unsigned char)new_g;
    p->b = (unsigned char)new_b;
}

/* ─────────────────────────────────────────────
 * [함수 3] 색상 반전 — 값 수정 가능
 * ───────────────────────────────────────────── */
void invert_pixel(/* 빈칸 4: brighten_pixel과 같은 형식 */)
{
    /* 빈칸 5: 각 채널 반전 */
    p->r = ___________;
    p->g = ___________;
    p->b = ___________;
}

/* ─────────────────────────────────────────────
 * [함수 4] 가장 밝은 픽셀의 주소 반환
 * ───────────────────────────────────────────── */
/* 빈칸 6: 반환 타입을 결정하세요
   힌트: 호출자가 반환된 포인터로 원본을 수정 못 하게 해야 함 */
______________ find_brightest(/* 빈칸 7: 픽셀 배열을 받되 수정 못 하게 */ pixels,
                              int count)
{
    /* 빈칸 8: 반복 안에서 비교 결과를 저장할 포인터
       힌트: 빈칸 6과 같은 타입의 포인터 */
    ______________ best = &pixels[0];
    int best_sum = pixels[0].r + pixels[0].g + pixels[0].b;

    for (int i = 1; i < count; i++) {
        int sum = pixels[i].r + pixels[i].g + pixels[i].b;
        if (sum > best_sum) {
            best_sum = sum;
            best = &pixels[i];
        }
    }
    return best;
}

/* ─────────────────────────────────────────────
 * 메인 함수
 * ───────────────────────────────────────────── */
int main(void)
{
    Pixel image[5] = {
        { 50,  100, 150},   // 어두운 파랑
        {200,   50,  50},   // 밝은 빨강
        {100,  100, 100},   // 회색
        {240,  240, 240},   // 거의 흰색
        { 30,  200,  30}    // 초록
    };

    /* (1) 모든 픽셀 출력 */
    printf("=== 원본 픽셀 ===\n");
    for (int i = 0; i < 5; i++) {
        printf("[%d] ", i);
        print_pixel(&image[i]);
    }

    /* (2) 첫 번째 픽셀에 밝기 +50 */
    printf("\n=== 픽셀[0]에 밝기 +50 ===\n");
    brighten_pixel(&image[0], 50);
    print_pixel(&image[0]);

    /* (3) 마지막 픽셀 색상 반전 */
    printf("\n=== 픽셀[4] 색상 반전 ===\n");
    invert_pixel(&image[4]);
    print_pixel(&image[4]);

    /* (4) 가장 밝은 픽셀 찾기 */
    printf("\n=== 가장 밝은 픽셀 ===\n");
    /* 빈칸 9: 빈칸 6과 같은 타입 */
    ______________ brightest = find_brightest(image, 5);
    print_pixel(brightest);

    /* 아래 줄의 주석을 풀면 컴파일 에러가 나야 정상! */
    /* brightest->r = 0; */

    return 0;
}
```

## 📌 임베디드 관점에서 (전기전자공학부 보너스)

여러분이 다룬 `const Pixel *` 패턴은 임베디드 시스템에서 매우 중요합니다.

예를 들어, **마이크로컨트롤러의 펌웨어 룩업 테이블** (예: 사인파 테이블, 폰트 데이터) 은:

```c
const unsigned char SINE_TABLE[256] = { /* ... */ };
```

이렇게 선언하면 컴파일러가 이 데이터를 **플래시 메모리(ROM)** 영역에 배치합니다. RAM 을 절약할 수 있죠. `const` 는 단순한 "수정 금지" 가 아니라, 컴파일러에게 **메모리 배치 힌트**까지 주는 강력한 키워드입니다.

다음 시간(분할 컴파일) 이후에 다룰 `volatile` 과 함께 자주 등장합니다.
