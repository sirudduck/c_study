## 0. 들어가기 — 왜 문자열은 따로 공부해야 하는가?

C에는 **`string` 타입이 없다**. 자바·파이썬과 다르게 C의 문자열은 **`char` 배열 + 끝을 알리는 `'\0'`** 이라는 약속에 불과하다.
즉, 9주차의 **배열**과 10주차의 **포인터**가 만나는 자리가 바로 11주차다. 두 주차의 개념이 비비고 들어와야 비로소 문자열이 손에 잡힌다.

> 🍳 **주방 비유 — "끝 표시 깃발"**
> 문자열은 일렬로 늘어놓은 글자 반찬통들이다. 어디까지가 음식인지(=문자열의 끝)를 알려주는 작은 **'\0' 깃발**이 마지막에 반드시 꽂혀 있어야 한다.
> 깃발이 없으면 다음 손님(printf)은 어디서 멈춰야 할지 모르고 **쓰레기 영역까지 계속 읽어버린다.**

## 1. 문자(char) vs 문자열(string) — 가장 먼저 구분하라

| 구분 | 표기 | 타입 | 메모리 |
|------|------|------|--------|
| 문자 상수 | `'A'` | `int`(C에서) / `char` | 1바이트 |
| 문자열 상수 | `"A"` | `char[2]` (= `'A'`, `'\0'`) | 2바이트 |
| 문자 변수 | `char c = 'A';` | `char` | 1바이트 |
| 문자열 변수 | `char s[] = "A";` | `char[2]` | 2바이트 |

> ⚠️ **시험 단골 함정**
> `'A'`와 `"A"`는 **다르다.** 작은따옴표는 문자 1개, 큰따옴표는 문자열(끝에 `'\0'` 포함).
> `sizeof('A')` → 보통 4 (C에서 문자 상수는 int 취급)
> `sizeof("A")` → 2

## 2. 문자열은 `char` 배열이다 — NULL 문자의 정체

### 2.1 핵심 약속

> **C 문자열 = `'\0'`로 끝나는 `char` 배열**

```c
char str[6] = {'H', 'e', 'l', 'l', 'o', '\0'};
char str[6] = "Hello";   // 위와 정확히 동일 (컴파일러가 '\0'을 자동 추가)
```

```
str[0] str[1] str[2] str[3] str[4] str[5]
 +-----+-----+-----+-----+-----+-----+
 |  H  |  e  |  l  |  l  |  o  | \0  |
 +-----+-----+-----+-----+-----+-----+
                                  ↑
                          끝을 알리는 깃발
```

### 2.2 ★ 시험 1순위 — 배열 크기는 글자 수 + 1

"Hello"라는 5글자 문자열을 담으려면 **최소 `char[6]`** 이 필요하다. `'\0'` 자리 1바이트를 반드시 추가.

| 문자열 | 글자 수 | 최소 배열 크기 |
|--------|---------|----------------|
| `"A"` | 1 | 2 |
| `"Hello"` | 5 | 6 |
| `"안녕"` (UTF-8) | 한글 1자 = 3byte → 6 | 7 |

### 2.3 NULL 문자 = `'\0'` = 정수 0

```c
'\0' == 0          // true
'\0' == NULL       // 책에서는 같다고 다루지만, NULL은 본래 포인터 상수
str[i] != '\0'     // 가장 표준적인 종료 검사
str[i] != 0        // 동일 (정수 0과 같으므로)
```

> 🍳 **암기 팁**: 종료 검사는 `'\0'`로 통일해 쓰자. `NULL`은 **포인터**가 비어있는지 검사할 때만 쓰는 것이 안전하다.

### 🛠️ 학생 실습 #1 — NULL 문자가 없으면?

> **문제:** 다음 코드의 출력이 왜 이상해지는지 설명하시오.

```c
#include <stdio.h>
int main(void) {
    char str[4];
    str[0] = 'a';
    str[1] = 'b';
    str[2] = 'c';
    // str[3] 자리에 '\0'을 넣지 않았다!
    printf("%s\n", str);
    return 0;
}
```

<details>
<summary>👉 해설</summary>

`printf("%s", ...)`는 `'\0'`을 만날 때까지 메모리를 계속 읽는다.
`str[3]`은 초기화되지 않은 쓰레기값이므로, 운이 좋으면 곧바로 0을 만나 멈추지만, 그렇지 않으면 **인접 메모리의 쓰레기까지 출력**한다. 최악의 경우 보호된 메모리 접근으로 **세그멘테이션 폴트**.

</details>

## 3. 문자 배열 초기화 — 시험 단골 비교

```c
char str[4] = { 'a', 'b', 'c', '\0' };  // ① 정석
char str[4] = "abc";                    // ② 가장 흔함 (자동 '\0')
char str[4] = "abcdef";                 // ③ ★ 오류! (크기 초과)
char str[6] = "abc";                    // ④ 가능, 뒤는 '\0'으로 채워짐
char str[4] = "";                       // ⑤ 빈 문자열, str[0] = '\0'
char str[]  = "abc";                    // ⑥ 컴파일러가 크기 자동 결정 (=4)
```

> ⚠️ **선언 이후의 대입은 불가!**
>
> ```c
> char str[10] = "Hello";
> str = "World";   // ❌ 컴파일 오류! 배열 이름은 변경 불가능한 주소
> strcpy(str, "World");  // ✅ 정답
> ```
>
> 배열 이름은 **포인터 상수**다 (10주차에서 본 그 약속). 통째로 다른 곳을 가리키게 만들 수 없다.

## 4. ★ 매우 중요 — 문자열 상수와 `char *p = "..."`

이 단원은 **메모리 영역**의 차이를 묻는 시험 단골 주제다.

### 4.1 두 가지 선언, 두 가지 메모리 영역

```c
char str[] = "Hello";    // ① 스택의 char 배열. 변경 가능
char *p    = "Hello";    // ② 텍스트 세그먼트의 상수. 변경 불가
```

```
[스택 영역]               [텍스트 세그먼트(읽기 전용)]
 ┌───────────────┐       ┌───────────────────────┐
 │ str: H e l l o\0 │   │ "Hello" (상수, 읽기전용) │
 └───────────────┘       └───────────────────────┘
                               ↑
                               │
                         p ────┘
```

### 4.2 시험 1순위 비교표

| 동작 | `char str[] = "Hello";` | `char *p = "Hello";` |
|------|------------------------|----------------------|
| `str[0] = 'h';` / `p[0] = 'h';` | ✅ 가능 | ❌ **런타임 오류** (텍스트 세그먼트 변경) |
| `str = "World";` / `p = "World";` | ❌ 컴파일 오류 | ✅ 가능 (p가 다른 상수를 가리킴) |
| `strcpy(str, "Bye");` / `strcpy(p, "Bye");` | ✅ 가능 | ❌ 런타임 오류 |

> 🍳 **암기 비유**
> - `char str[]` = "내 반찬통" → 내용물 마음대로 바꿔도 됨, 통은 그 자리에 고정
> - `char *p`   = "남의 집 가리키는 라벨" → 라벨이 가리키는 곳을 바꿀 수는 있지만, 그 집 내용은 못 만짐

---

### 🛠️ 학생 실습 #2 — 메모리 영역 함정

> **문제:** 다음 중 컴파일은 되지만 **실행 시 크래시**하는 것을 모두 고르시오.

```c
// (가)
char a[] = "abc";
a[0] = 'X';

// (나)
char *b = "abc";
b[0] = 'X';

// (다)
char *c = "abc";
c = "xyz";
```

<details>
<summary>👉 정답: (나)</summary>

- (가) `a`는 스택의 char 배열. ✅ 안전.
- (나) `b`는 텍스트 세그먼트 문자열 상수를 가리킨다. `b[0]` 변경은 **읽기전용 영역 침범** → 크래시.
- (다) `c`가 가리키는 **포인터 자체**의 값을 바꾸는 것이므로 ✅ 안전.

</details>

## 5. 문자 입출력 라이브러리 (`stdio.h`, `conio.h`)

| 함수 | 헤더 | 버퍼 사용 | 에코(echo) | 비고 |
|------|------|-----------|------------|------|
| `getchar()` | `stdio.h` | ✅ (엔터까지 대기) | ✅ | 표준 |
| `putchar(c)` | `stdio.h` | ✅ | — | 표준 |
| `_getch()` | `conio.h` | ❌ (즉시) | ❌ | Windows 한정 |
| `_getche()` | `conio.h` | ❌ | ✅ | Windows 한정 |
| `_putch(c)` | `conio.h` | ❌ | — | Windows 한정 |
| `scanf("%c", &c)` | `stdio.h` | ✅ | ✅ | 공백 처리 주의 |
| `printf("%c", c)` | `stdio.h` | ✅ | — | |

### 5.1 EOF — 입력의 끝

```c
int ch;                              // ★ char 아닌 int!
while ((ch = getchar()) != EOF) {
    putchar(ch);
}
```

> ⚠️ **시험 단골 — 왜 `int`인가?**
> `EOF`는 보통 `-1`로 정의된 **정수 상수**다. 반환을 `char`로 받으면 `-1`(0xFF)이 정상 문자 0xFF와 구별되지 않을 수 있다. 그래서 반드시 **`int`로 받는다.**

> **키보드에서 EOF 입력:** Windows는 `Ctrl + Z` 후 엔터, Linux/Mac은 `Ctrl + D`.

## 6. 문자열 입출력 라이브러리

| 함수 | 용도 | 공백 포함? |
|------|------|-----------|
| `scanf("%s", s)` | 한 단어 입력 | ❌ (공백/엔터에서 멈춤) |
| `gets_s(s, sizeof(s))` | 한 줄 입력 | ✅ |
| `printf("%s", s)` | 한 단어 출력 | — |
| `puts(s)` | 한 줄 출력 + 줄바꿈 자동 추가 | — |

> ⚠️ **시험 1순위**
> - `scanf("%s", name);` — **`&` 붙이지 않는다!** 배열 이름이 이미 주소이기 때문.
> - 한 줄 전체를 받으려면 반드시 `gets_s()` 사용.
> - 본래 `gets()`는 버퍼 오버플로 위험으로 **C11에서 제거**됨. 대체로 `gets_s()` 또는 `fgets()` 사용.

### 6.1 함정 예제

```c
char name[100], address[100];
printf("이름: ");        scanf("%s", name);
printf("주소: ");        scanf("%s", address);

// 입력: "홍길동" / "서울시 종로구 1번지"
// 결과: address에는 "서울시"까지만 저장된다. (공백에서 끊김)
```

해결: `gets_s(address, sizeof(address));`

## 7. 문자 처리 함수 (`<ctype.h>`)

### 7.1 검사 함수 (참이면 0이 아닌 값 반환)

| 함수 | 검사 대상 |
|------|----------|
| `isalpha(c)` | 영문자 (a~z, A~Z) |
| `isupper(c)` | 대문자 |
| `islower(c)` | 소문자 |
| `isdigit(c)` | 숫자 (0~9) |
| `isalnum(c)` | 영문자 또는 숫자 |
| `isxdigit(c)` | 16진수 숫자 (0~9, A~F, a~f) |
| `isspace(c)` | 공백 문자 (`' '`, `\n`, `\t`, `\v`, `\r`) |
| `ispunct(c)` | 구두점 |
| `isprint(c)` | 출력 가능 문자 |
| `iscntrl(c)` | 제어 문자 |

### 7.2 변환 함수

| 함수 | 변환 |
|------|------|
| `toupper(c)` | 소문자 → 대문자 |
| `tolower(c)` | 대문자 → 소문자 |

```c
#include <ctype.h>
int c;
while ((c = getchar()) != EOF) {
    if (islower(c)) c = toupper(c);
    putchar(c);
}
// abc → ABC
```

## 8. 문자열 처리 함수 (`<string.h>`) — ★ 시험 핵심

| 함수 | 기능 | 반환 |
|------|------|------|
| `strlen(s)` | 길이(끝 `'\0'` 제외) | `size_t` |
| `strcpy(dst, src)` | 복사 | `dst` |
| `strncpy(dst, src, n)` | 최대 n자 복사 | `dst` |
| `strcat(dst, src)` | 이어붙이기 | `dst` |
| `strncat(dst, src, n)` | 최대 n자 이어붙이기 | `dst` |
| `strcmp(s1, s2)` | 사전순 비교 | 음수/0/양수 |
| `strncmp(s1, s2, n)` | 앞 n자 비교 | 음수/0/양수 |
| `strchr(s, c)` | 문자 검색 | 위치 포인터 / NULL |
| `strstr(s1, s2)` | 부분 문자열 검색 | 위치 포인터 / NULL |
| `strtok(s, delim)` | 토큰 분리 | 토큰 시작 포인터 / NULL |

### 8.1 `strcmp` — ★ 결과 해석에 주의

```c
int r = strcmp(s1, s2);
// r <  0  : s1이 사전순으로 앞 (예: "cat" vs "dog")
// r == 0  : 동일
// r >  0  : s1이 사전순으로 뒤
```

> ⚠️ **시험 함정:** "두 문자열이 같은가?" 검사를 `if (s1 == s2)`로 쓰면 **포인터 비교**가 된다. 반드시 `strcmp(s1, s2) == 0` 사용.

### 8.2 `strcpy`/`strcat` — 버퍼 크기 책임은 호출자에게

```c
char dst[6] = "Hello";
strcat(dst, "World");   // ❌ dst 크기 부족 → 버퍼 오버플로!

char dst2[20] = "Hello";
strcat(dst2, "World");  // ✅
```

> 💡 **안전한 대안:** `strncpy`, `strncat`, 또는 C11의 `strcpy_s`, `strcat_s`.

### 8.3 `strtok` — 토큰 분리

```c
char s[] = "Man is immortal, because he has a soul";
char seps[] = " ,\t\n";
char *tok = strtok(s, seps);          // 첫 토큰
while (tok != NULL) {
    printf("토큰: %s\n", tok);
    tok = strtok(NULL, seps);         // ★ 두 번째부터는 NULL을 전달
}
```

> ⚠️ `strtok`은 **원본 문자열을 파괴**한다(구분자 자리에 `'\0'` 삽입). 보존이 필요하면 복사 후 사용.

### 🛠️ 학생 실습 #3 — `strlen` 직접 구현

> **문제:** `strlen`을 포인터 산술로 구현하시오.

<details>
<summary>👉 정답 예시</summary>

```c
#include <stdio.h>

size_t my_strlen(const char *s) {
    const char *p = s;
    while (*p != '\0') p++;
    return p - s;        // 포인터 차이 = 원소 개수
}

int main(void) {
    printf("%zu\n", my_strlen("Hello, World!"));   // 13
    return 0;
}
```

**시험 포인트:** 같은 배열 안에서 포인터끼리 뺄셈 = 원소 개수. (10주차 복습)

</details>

## 9. 문자열 ↔ 수치 변환

### 9.1 `sprintf` / `sscanf` (만능)

```c
// 수치 → 문자열
char buf[50];
int x = 10, y = 20;
sprintf(buf, "%d + %d = %d", x, y, x + y);
// buf: "10 + 20 = 30"

// 문자열 → 수치
char str[] = "file 12";
char name[10];
int num;
sscanf(str, "%s %d", name, &num);
// name="file", num=12
```

### 9.2 전용 변환 함수 (`<stdlib.h>`)

| 함수 | 변환 |
|------|------|
| `atoi("100")` | 문자열 → `int` |
| `atof("12.93")` | 문자열 → `double` |
| `atol("123456")` | 문자열 → `long` |

```c
#include <stdlib.h>
int    i = atoi("100");      // 100
double d = atof("12.93");    // 12.93
```

> 💡 `printf` vs `sprintf` — 출력 대상이 화면이면 `printf`, **문자 버퍼**면 `sprintf`. 한 글자(s) 차이.

## 10. 문자열 배열 (여러 개의 문자열 저장)

### 10.1 2차원 char 배열

```c
char menu[5][10] = {
    "init",
    "open",
    "close",
    "read",
    "write"
};
// 메모리: 모든 행이 10바이트씩 고정 → 낭비 가능
```

```
menu[0]: i n i t \0 . . . . .
menu[1]: o p e n \0 . . . . .
menu[2]: c l o s e \0 . . . .
```

### 10.2 char* 포인터 배열

```c
char *menu[] = {
    "init", "open", "close", "read", "write"
};
// 각 문자열은 텍스트 세그먼트, 배열에는 주소만 저장 → 메모리 효율 ↑
// 단, 각 문자열은 변경 불가 (4단원 규칙)
```

| 구분 | `char arr[N][M]` | `char *arr[N]` |
|------|------------------|----------------|
| 메모리 | N×M 바이트 고정 | 포인터 N개 + 각 문자열 |
| 내용 변경 | ✅ | ❌ (텍스트 세그먼트) |
| 다른 문자열로 교체 | ❌ (`arr[i] = "..."` 불가) | ✅ (`arr[i] = "..."` 가능) |

## 11. 미니 프로젝트 핵심 패턴 정리

PDF의 실습/미니 프로젝트에 등장하는 핵심 알고리즘만 추려둔다.

### 11.1 단어 개수 세기 — 상태 기계 패턴

```c
int count_word(const char *s) {
    int wc = 0;
    int in_word = 0;             // 현재 단어 안에 있는가?
    for (int i = 0; s[i] != '\0'; i++) {
        if (isalpha((unsigned char)s[i])) {
            if (!in_word) { wc++; in_word = 1; }
        } else {
            in_word = 0;
        }
    }
    return wc;
}
```

> 🍳 **핵심:** "공백 다음 첫 알파벳에서만" 카운트. 단어 중간에서는 카운트하지 않기 위해 `in_word` 플래그 사용.

### 11.2 비밀번호 강도 검증 — 다중 카운터 패턴

```c
int lower = 0, upper = 0, digit = 0;
for (int i = 0; pass[i] != '\0'; i++) {
    if (islower(pass[i])) lower++;
    if (isupper(pass[i])) upper++;
    if (isdigit(pass[i])) digit++;
}
int strong = (strlen(pass) >= 7) && lower && upper && digit;
```

### 11.3 시저 암호화 — 모듈러 시프트 패턴

```c
void encrypt(char *s, int shift) {
    for (int i = 0; s[i] != '\0'; i++) {
        if (s[i] >= 'a' && s[i] <= 'z') {
            s[i] = (s[i] - 'a' + shift) % 26 + 'a';
        } else if (s[i] >= 'A' && s[i] <= 'Z') {
            s[i] = (s[i] - 'A' + shift) % 26 + 'A';
        }
    }
}
```

> ⚠️ **PDF 코드의 문제점:** PDF는 `if (cipher[i] > 'z') cipher[i] -= 26;` 으로 단순 처리하는데, `shift`가 26 이상이거나 음수일 때 깨진다. `% 26` 으로 통일하는 것이 안전.

## 12. 시험 직전 체크리스트 ✅

- [ ] `'A'` vs `"A"` 차이 (1바이트 vs 2바이트)
- [ ] `'\0'`의 ASCII 값은 **0**
- [ ] "Hello" 저장에 필요한 최소 배열 크기는 **6**
- [ ] `char s[] = "Hello"` 와 `char *p = "Hello"`의 **메모리 영역**과 **변경 가능 여부**
- [ ] `scanf("%s", name)` 에서 **`&` 안 붙임**
- [ ] `gets_s` vs `scanf("%s")` — 공백 포함 여부
- [ ] `getchar()`는 반환값을 **`int`로** 받기 (EOF 처리)
- [ ] `strcmp` 결과: 같으면 **0**, s1이 앞이면 **음수**
- [ ] 문자열 비교는 `==` 가 아니라 **`strcmp`**
- [ ] `strcat` 전 dst 버퍼 크기 확인 (오버플로 방지)
- [ ] `strtok`은 두 번째부터 **`NULL`** 전달
- [ ] `ctype.h` 함수들: `isalpha`, `isdigit`, `isupper`, `tolower`, `toupper`
- [ ] `sprintf` / `sscanf` / `atoi` / `atof` 용도
- [ ] 문자열 배열: `char arr[N][M]` vs `char *arr[N]`
