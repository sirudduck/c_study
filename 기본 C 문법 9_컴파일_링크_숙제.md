## 1. 예제 프로젝트: 학생 성적 관리 시스템

아래와 같이 4개의 역할로 파일을 나눈 프로젝트를 생각해 보자.

```
project/
├── main.c          ← 프로그램 진입점
├── student.h / .c  ← 학생 구조체와 기본 함수
├── grade.h / .c    ← 성적 계산 함수
└── report.h / .c   ← 출력/리포트 함수
```

각 파일의 내용은 다음과 같다.

**student.h** — 학생 구조체 정의 + 기본 함수 선언
```c
#ifndef STUDENT_H
#define STUDENT_H

typedef struct {
    char  name[20];
    int   id;
    float score;
} Student;

Student create_student(const char *name, int id, float score);

#endif
```

**grade.h** — 성적 계산 함수 선언 (Student 타입이 필요하므로 student.h를 포함)
```c
#ifndef GRADE_H
#define GRADE_H

#include "student.h"    /* ← Student 타입을 알아야 하므로 */

float    calc_average(Student *arr, int n);
Student *find_top(Student *arr, int n);
char     get_letter_grade(float score);

#endif
```

**report.h** — 출력 함수 선언 (Student 타입이 필요하므로 student.h를 포함)
```c
#ifndef REPORT_H
#define REPORT_H

#include "student.h"    /* ← Student 타입을 알아야 하므로 */

void print_all(Student *arr, int n);
void print_report(Student *arr, int n, float avg);

#endif
```

**main.c** — 모든 기능을 조합
```c
#include <stdio.h>
#include "student.h"
#include "grade.h"
#include "report.h"

int main(void) {
    Student class_a[3] = { ... };
    float avg = calc_average(class_a, 3);
    print_report(class_a, 3, avg);
    return 0;
}
```

## 2. 파일 간 #include 관계도

### 2.1 헤더 파일끼리의 의존 관계

```
                    ┌──────────────┐
                    │  student.h   │
                    │              │
                    │  Student 구조체 │
                    │  create_student │
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
              ▼            ▼            ▼
       ┌────────────┐ ┌────────────┐ ┌────────────┐
       │  grade.h   │ │ report.h   │ │  main.c    │
       │            │ │            │ │            │
       │ #include   │ │ #include   │ │ #include   │
       │ "student.h"│ │ "student.h"│ │ "student.h"│
       │            │ │            │ │ "grade.h"  │
       │ calc_avg   │ │ print_all  │ │ "report.h" │
       │ find_top   │ │ print_rpt  │ │            │
       │ get_letter │ │            │ │ main()     │
       └────────────┘ └────────────┘ └────────────┘
```

`student.h`는 **세 곳**에서 include된다.
- `grade.h`가 include (Student 타입 필요)
- `report.h`가 include (Student 타입 필요)
- `main.c`가 직접 include

> **인클루드 가드가 없다면?**  
> `main.c`는 `student.h`를 직접 1번 + `grade.h` 경유 1번 + `report.h` 경유 1번  
> = **총 3번** 포함하게 되어 `Student` 구조체가 3번 정의되고 오류가 발생한다.  
> `#ifndef STUDENT_H` 가드 덕분에 첫 번째 이후는 무시된다.

### 2.2 main.c의 전체 include 트리

`main.c`가 직접·간접으로 포함하는 헤더를 트리로 그리면:

```
main.c
├── #include <stdio.h>          ← 시스템 헤더
├── #include "student.h"        ← Student 구조체, create_student 선언
├── #include "grade.h"
│   └── #include "student.h"    ← ⚠ 이미 위에서 포함됨 → 인클루드 가드가 무시 처리
└── #include "report.h"
    └── #include "student.h"    ← ⚠ 이미 위에서 포함됨 → 인클루드 가드가 무시 처리
```

## 3. #include는 복사-붙여넣기다

전처리기(조교)가 `main.c`의 `#include`를 처리하면,  
**각 헤더 파일의 내용이 그 자리에 그대로 복사**된다.

### 3.1 처리 전: 우리가 작성한 main.c

```c
#include <stdio.h>       /* ── ① */
#include "student.h"     /* ── ② */
#include "grade.h"       /* ── ③ */
#include "report.h"      /* ── ④ */

int main(void) {
    Student class_a[3] = { ... };
    float avg = calc_average(class_a, 3);
    print_report(class_a, 3, avg);
    return 0;
}
```

### 3.2 처리 과정: 한 줄씩 복사-붙여넣기

**① `#include <stdio.h>` 처리:**

```c
/* ══════════ ① stdio.h 내용이 이 자리에 복사됨 ══════════ */
int printf(const char *format, ...);
int scanf(const char *format, ...);
typedef struct _FILE FILE;
/* ... (실제로는 수천 줄) ... */
/* ══════════ ① stdio.h 끝 ══════════════════════════════ */

#include "student.h"     /* ── ② 아직 처리 안 됨 */
#include "grade.h"       /* ── ③ 아직 처리 안 됨 */
#include "report.h"      /* ── ④ 아직 처리 안 됨 */

int main(void) { ... }
```

**② `#include "student.h"` 처리:**

```c
/* ① stdio.h 내용 (생략) */

/* ══════════ ② student.h 내용이 이 자리에 복사됨 ══════════ */
/* #ifndef STUDENT_H → 아직 정의 안 됨 → 통과 */
/* #define STUDENT_H → 지금 정의! */
typedef struct {
    char  name[20];
    int   id;
    float score;
} Student;
Student create_student(const char *name, int id, float score);
/* #endif */
/* ══════════ ② student.h 끝 ═════════════════════════════ */

#include "grade.h"       /* ── ③ 아직 처리 안 됨 */
#include "report.h"      /* ── ④ 아직 처리 안 됨 */

int main(void) { ... }
```

**③ `#include "grade.h"` 처리:**

grade.h 안에 `#include "student.h"`가 있으므로, 그것도 함께 처리된다.

```c
/* ① stdio.h 내용 (생략) */
/* ② student.h 내용 (위에서 이미 복사됨) */

/* ══════════ ③ grade.h 내용이 이 자리에 복사됨 ═══════════ */
/* #ifndef GRADE_H → 아직 정의 안 됨 → 통과 */
/* #define GRADE_H → 지금 정의! */

    /* grade.h 안의 #include "student.h" 처리 시도: */
    /* #ifndef STUDENT_H → ⚠ 이미 ②에서 정의됨! → 전체 건너뜀 */
    /* (student.h 내용이 여기에 복사되지 않음) */

float    calc_average(Student *arr, int n);
Student *find_top(Student *arr, int n);
char     get_letter_grade(float score);
/* #endif */
/* ══════════ ③ grade.h 끝 ════════════════════════════════ */

#include "report.h"      /* ── ④ 아직 처리 안 됨 */

int main(void) { ... }
```

> **핵심 포인트:** grade.h 안의 `#include "student.h"`가 실행되었지만,  
> `STUDENT_H`가 이미 ②에서 정의되었으므로 **아무것도 복사되지 않는다.**  
> 이것이 인클루드 가드의 역할이다.

**④ `#include "report.h"` 처리:**

report.h도 마찬가지로 `#include "student.h"`가 무시된다.

```c
/* ① stdio.h 내용 (생략) */
/* ② student.h 내용 (위에서 이미 복사됨) */
/* ③ grade.h 내용 (위에서 이미 복사됨) */

/* ══════════ ④ report.h 내용이 이 자리에 복사됨 ══════════ */
/* #ifndef REPORT_H → 아직 정의 안 됨 → 통과 */
/* #define REPORT_H → 지금 정의! */

    /* report.h 안의 #include "student.h" 처리 시도: */
    /* #ifndef STUDENT_H → ⚠ 이미 정의됨! → 전체 건너뜀 */

void print_all(Student *arr, int n);
void print_report(Student *arr, int n, float avg);
/* #endif */
/* ══════════ ④ report.h 끝 ═══════════════════════════════ */

int main(void) {
    Student class_a[3] = { ... };
    float avg = calc_average(class_a, 3);
    print_report(class_a, 3, avg);
    return 0;
}
```

### 3.3 전처리 완료 후: 교관(컴파일러)이 보는 최종 모습

```c
/* ── stdio.h에서 온 내용 ──────────────────────────── */
int printf(const char *format, ...);
int scanf(const char *format, ...);
/* ... */

/* ── student.h에서 온 내용 (1번만 복사됨) ──────────── */
typedef struct {
    char  name[20];
    int   id;
    float score;
} Student;
Student create_student(const char *name, int id, float score);

/* ── grade.h에서 온 내용 ──────────────────────────── */
float    calc_average(Student *arr, int n);
Student *find_top(Student *arr, int n);
char     get_letter_grade(float score);

/* ── report.h에서 온 내용 ─────────────────────────── */
void print_all(Student *arr, int n);
void print_report(Student *arr, int n, float avg);

/* ── 원래 main.c 코드 ────────────────────────────── */
int main(void) {
    Student class_a[3] = { ... };
    float avg = calc_average(class_a, 3);
    print_report(class_a, 3, avg);
    return 0;
}
```

`#include`는 모두 사라지고, 모든 선언이 **하나의 긴 파일**로 합쳐졌다.  
교관(컴파일러)은 이 상태에서 기계어로 변환을 시작한다.

## 4. .c 파일끼리는 서로 모른다

여기서 중요한 점이 있다.  
**`.c` 파일은 서로를 직접 include하지 않는다.**

```
         ┌─────────────────────────────────────────┐
         │            헤더 파일의 세계              │
         │                                         │
         │         ┌──────────┐                    │
         │         │student.h │                    │
         │         └────┬─────┘                    │
         │         ┌────┴─────┐                    │
         │    ┌────┴───┐ ┌───┴────┐               │
         │    │grade.h │ │report.h│               │
         │    └────────┘ └────────┘               │
         └─────────────────────────────────────────┘
              ↑ include    ↑ include    ↑ include
              │            │            │
         ┌────┴───┐  ┌────┴───┐  ┌────┴────┐
         │grade.c  │  │report.c│  │ main.c  │
         └─────────┘  └────────┘  └─────────┘
              ✗ ──────── ✗ ──────── ✗
                .c끼리는 연결 없음!
```

각 `.c` 파일은:
- 자신이 필요한 `.h` 파일만 include한다.
- 다른 `.c` 파일의 존재를 **전혀 모른다.**
- 독립적으로 컴파일되어 각각 `.obj` 파일이 된다.

### 4.1 각 .c 파일이 보는 세상

```
┌─ grade.c가 아는 것 ──────────────────────────────────┐
│                                                       │
│  #include "grade.h"  → Student 구조체 (student.h 경유)│
│                        calc_average 선언               │
│                        find_top 선언                   │
│                        get_letter_grade 선언           │
│  #include <stdio.h>  → printf 선언                    │
│                                                       │
│  ✗ report.c에 뭐가 있는지 모름                        │
│  ✗ main.c에 뭐가 있는지 모름                          │
└───────────────────────────────────────────────────────┘

┌─ report.c가 아는 것 ─────────────────────────────────┐
│                                                       │
│  #include "report.h" → Student 구조체 (student.h 경유)│
│                        print_all 선언                  │
│                        print_report 선언               │
│  #include <stdio.h>  → printf 선언                    │
│                                                       │
│  ✗ grade.c에 뭐가 있는지 모름                         │
│  ✗ main.c에 뭐가 있는지 모름                          │
└───────────────────────────────────────────────────────┘

┌─ main.c가 아는 것 ───────────────────────────────────┐
│                                                       │
│  #include "student.h" → Student 구조체, create_student │
│  #include "grade.h"   → calc_average, find_top 선언   │
│  #include "report.h"  → print_all, print_report 선언  │
│  #include <stdio.h>   → printf 선언                   │
│                                                       │
│  ✗ grade.c의 함수 본문이 뭔지 모름                    │
│  ✗ report.c의 함수 본문이 뭔지 모름                   │
│  → "이런 함수가 있다"는 선언만 알 뿐,                 │
│    실제 코드가 어디 있는지는 모른다                    │
└───────────────────────────────────────────────────────┘
```

## 5. 그래서 링커가 필요하다

각 `.c` 파일은 독립적으로 컴파일되어 `.obj` 파일(수료증)이 된다.  
하지만 `.obj` 파일에는 **빈칸(???)** 이 남아있다.

```
┌─ main.obj ──────────────────┐
│                              │
│  main():                     │
│    CALL calc_average @ ???   │──── "grade.obj 어딘가에 있을 거야"
│    CALL print_report @ ???   │──── "report.obj 어딘가에 있을 거야"
│    CALL printf       @ ???   │──── "stdio.lib 어딘가에 있을 거야"
│                              │
└──────────────────────────────┘

┌─ grade.obj ─────────────────┐
│                              │
│  calc_average():  0x00401000 │  ← 실제 주소 있음
│  find_top():      0x00401080 │  ← 실제 주소 있음
│  get_letter_grade: 0x00401100│  ← 실제 주소 있음
│                              │
│  CALL printf      @ ???      │──── "stdio.lib 어딘가에 있을 거야"
│                              │
└──────────────────────────────┘

┌─ report.obj ────────────────┐
│                              │
│  print_all():     0x00402000 │  ← 실제 주소 있음
│  print_report():  0x00402080 │  ← 실제 주소 있음
│                              │
│  CALL printf      @ ???      │──── "stdio.lib 어딘가에 있을 거야"
│  CALL calc_average @ ???     │──── "grade.obj 어딘가에 있을 거야"
│                              │
└──────────────────────────────┘

┌─ student.obj ───────────────┐
│                              │
│  create_student(): 0x00403000│  ← 실제 주소 있음
│                              │
└──────────────────────────────┘
```

졸업 심사관(링커)이 이 수료증들을 모아서 빈칸을 채운다:

```
졸업 심사관(링커)의 작업:

main.obj의    calc_average @ ???   →  0x00401000  (grade.obj에서 찾음 ✓)
main.obj의    print_report @ ???   →  0x00402080  (report.obj에서 찾음 ✓)
main.obj의    printf       @ ???   →  0x7FFA2300  (stdio.lib에서 찾음 ✓)
grade.obj의   printf       @ ???   →  0x7FFA2300  (stdio.lib에서 찾음 ✓)
report.obj의  printf       @ ???   →  0x7FFA2300  (stdio.lib에서 찾음 ✓)
report.obj의  calc_average @ ???   →  0x00401000  (grade.obj에서 찾음 ✓)

                    모든 빈칸 채움 완료!
                           ↓
                    ┌──────────────┐
                    │ program.exe  │
                    │ (완성된 요리사) │
                    └──────────────┘
```

## 6. 전체 흐름 한눈에 보기

```
                    [헤더의 세계 — 선언만]
                    ┌──────────┐
                    │student.h │ ← Student 구조체
                    └──┬───┬───┘
                 ┌─────┘   └─────┐
                 ▼               ▼
            ┌────────┐     ┌─────────┐
            │grade.h │     │report.h │
            └────────┘     └─────────┘


    [전처리 — 조교가 #include를 복사-붙여넣기]

    student.c          grade.c          report.c          main.c
    #include           #include         #include          #include
    "student.h"        "grade.h"        "report.h"        "student.h"
                       → student.h      → student.h       "grade.h"
                         자동 포함        자동 포함         "report.h"
        │                  │                │                 │
        ▼                  ▼                ▼                 ▼
    student.i          grade.i          report.i          main.i
    (모든 선언이        (모든 선언이      (모든 선언이       (모든 선언이
     합쳐진 상태)        합쳐진 상태)      합쳐진 상태)       합쳐진 상태)


    [컴파일 — 교관이 각각 독립적으로 기계어 변환]

    student.i          grade.i          report.i          main.i
        │                  │                │                 │
        ▼                  ▼                ▼                 ▼
    student.obj        grade.obj        report.obj        main.obj
    (빈칸 없음)         (printf ???)     (printf ???)      (calc_avg ???)
                                        (calc_avg ???)    (print_rpt ???)
                                                          (printf ???)

         ※ .obj끼리는 서로의 존재를 모른다!


    [링킹 — 심사관이 빈칸을 채워서 하나로 합침]

    student.obj ──┐
    grade.obj   ──┤
    report.obj  ──┼──→  졸업 심사관(링커)  ──→  program.exe
    main.obj    ──┤                              (완성된 요리사)
    stdio.lib   ──┘


    [실행 — 주방장(OS)이 고용]

    program.exe  ──→  주방장(OS)이 조리대(RAM)에 올림  ──→  실행!
```

## 정리

```
[헤더 파일 (.h)]
  - 서로 include할 수 있다 (grade.h가 student.h를 include)
  - 인클루드 가드(#ifndef)가 중복 복사를 방지한다
  - #include는 말 그대로 복사-붙여넣기다

[소스 파일 (.c)]
  - .h만 include한다. 다른 .c를 include하지 않는다.
  - 각각 독립적으로 컴파일되어 .obj가 된다.
  - 다른 .c에 어떤 코드가 있는지 전혀 모른다.

[링커]
  - .obj끼리 서로 모르는 상태에서,
    빈칸(???)을 채워 하나의 .exe로 합치는 역할을 한다.
```
